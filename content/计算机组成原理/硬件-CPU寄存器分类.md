# 硬件-CPU寄存器分类

[[硬件-CPU]]内部的寄存器根据其功能和对程序员的可见性，可以分为两大类：**用户可见寄存器**和**控制与状态寄存器**。

## 1. 用户可见寄存器 (User-Visible Registers)

这类寄存器允许程序员（通常是汇编语言程序员）直接通过指令进行访问和操作。

- **通用寄存器 (General-Purpose Registers)**：功能灵活，既可以用于存放数据，也可以用于存放地址（如在寄存器间接寻址、基址寻址、变址寻址中）。
- **数据寄存器 (Data Registers)**：专门用于存放操作数和运算结果。
- **地址寄存器 (Address Registers)**：专门用于存放地址，如段指针、变址寄存器、堆栈指针。
- **条件码寄存器 (Condition Code Register)**：也称标志寄存器，存放ALU运算后产生的状态标志，如零标志（Z）、负标志（N）、溢出标志（V）、进位标志（C）。

## 2. 控制和状态寄存器 (Control and Status Registers)

这类寄存器由[[硬件-控制器]]使用，以控制CPU的操作，对普通用户程序通常是不可见的（透明的），但可以被操作系统等特权程序访问。

- **[[硬件-寄存器-MAR]] (存储器地址寄存器)**：存放访存地址。
- **[[硬件-寄存器-MDR]] (存储器数据寄存器)**：存放与内存交换的数据。
- **[[硬件-寄存-PC]] (程序计数器)**：存放下一条指令的地址。
- **[[硬件-寄存器-IR]] (指令寄存器)**：存放当前执行的指令。
- **程序状态字 (Program Status Word, PSW)**：一个集合了各种状态信息（如条件码、中断屏蔽位、特权级等）的寄存器。

这种分类清晰地划分了程序员编程接口和CPU内部控制的界限。
