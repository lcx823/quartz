# 概念-编译程序

编译程序（Compiler）是一种**语言处理程序**，属于[[概念-软件|系统软件]]。它的核心功能是将程序员用高级语言（如C++、Pascal、FORTRAN）编写的**源程序（Source Program）**，**一次性地**、**完整地**翻译成计算机可以直接执行的**机器语言程序（目标程序, Object Program）**。

## 工作流程

1.  **输入**：接收整个源程序文件。
2.  **翻译**：对源程序进行词法分析、语法分析、语义分析、优化等一系列步骤，最终生成等效的机器语言代码。
3.  **输出**：生成一个独立的目标程序文件（如 `.obj` 或 `.exe` 文件）。
4.  **执行**：用户之后可以多次运行这个目标程序，而无需再次进行翻译。

## 核心特点

- **一次性翻译**：整个翻译过程和执行过程是完全分离的。先完整翻译，后独立执行。
- **高效率执行**：一旦编译完成，目标程序可以直接在硬件上运行，执行速度快。
- **调试不便**：如果程序出错，需要重新修改源程序，再次编译整个程序，调试周期较长。
- **平台依赖**：编译生成的目标程序通常依赖于特定的[[框架-指令系统]]和操作系统。

> 编译程序是将用户编写的高级语言程序(源程序)的全部语句一次全部翻译成机器语言程序,而后再执行机器语言程序。因此,只要源程序不变,就无须再次进行翻译。
