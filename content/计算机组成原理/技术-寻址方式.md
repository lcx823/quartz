# 技术-寻址方式

寻址方式（Addressing Mode）是指确定指令中操作数地址以及下一条指令地址的方法。它定义了指令中地址码字段的解释方式。

- **形式地址 (A)**：指令中给出的地址。
- **有效地址 (EA)**：操作数的真实地址。

## 数据寻址方式

1.  **立即寻址**
    - **原理**：指令的地址码字段直接存放**操作数本身**，而非地址。
    - **特点**：取指后无需再次访存，执行速度快，但操作数大小受地址码位数限制。

2.  **直接寻址**
    - **原理**：指令的地址码字段存放的是操作数的**有效地址**。`EA = A`。
    - **特点**：简单直观，但寻址范围受地址码位数限制。

3.  **间接寻址**
    - **原理**：指令的地址码字段存放的是有效地址的**地址**。`EA = (A)`。
    - **特点**：扩大了寻址范围，但需要多次访存，执行速度慢。

4.  **寄存器寻址**
    - **原理**：地址码字段存放的是存放操作数的**寄存器编号**。`EA = Ri`。
    - **特点**：无需访存，执行速度极快。

5.  **寄存器间接寻址**
    - **原理**：地址码字段指定的寄存器中存放的是操作数的**有效地址**。`EA = (Ri)`。
    - **特点**：比间接寻址快（少一次访存），同时保持了寻址的灵活性。

6.  **基址寻址**
    - **原理**：有效地址由指令中的形式地址与**基址寄存器（BR）**的内容相加得到。`EA = A + (BR)`。
    - **特点**：便于程序重定位和多道程序设计，扩大了寻址范围。

7.  **变址寻址**
    - **原理**：有效地址由指令中的形式地址与**变址寄存器（IX）**的内容相加得到。`EA = A + (IX)`。
    - **特点**：非常适合处理数组和循环程序，形式地址A通常是数组首地址，变址寄存器内容是偏移量。

8.  **相对寻址**
    - **原理**：有效地址由**程序计数器（PC）**的内容与指令中的形式地址（位移量）相加得到。`EA = (PC) + A`。
    - **特点**：常用于转移指令，生成的地址是相对于当前指令的，便于程序在内存中浮动。

> 数据寻址方式种类较多,在指令字中必须设一字段来指明属于哪一种寻址方式。指令的地址码字段通常都不代表操作数的真实地址,故把它称为形式地址,记作A。操作数的真实地址称为有效地址,记作EA,它是由寻址方式和形式地址共同来确定的。
