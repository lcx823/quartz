# 技术-RISC技术

RISC（Reduced Instruction Set Computer，精简指令系统计算机）是一种计算机体系结构的设计思想，与之相对的是CISC（Complex Instruction Set Computer，复杂指令系统计算机）。RISC的核心思想是通过简化指令系统来提高处理器的执行效率。

## 产生背景

对传统CISC体系结构的研究发现，程序中约80%的时间在使用约20%的简单指令。复杂的指令不仅使用频率低，而且增加了[[硬件-控制器]]的复杂性，拖慢了所有指令（包括简单指令）的执行速度。RISC技术应运而生，旨在用这些常用的简单指令组合来实现复杂功能。

> 人们从80-20规律中得到启示:能否仅仅用最常用的20%的简单指令,重新组合不常用的80%的指令功能呢?这便引发出RISC技术。

## 主要特征

1.  **指令数量少**：只选取使用频率最高的简单指令。
2.  **指令长度固定**：所有指令长度相同，便于译码和[[技术-指令流水|流水线]]处理。
3.  **指令格式和寻址方式少**：简化了指令格式和寻址方式的种类。
4.  **Load/Store结构**：只有取数（LOAD）和存数（STORE）指令可以访问内存，所有其他算术逻辑操作都在寄存器之间进行。
5.  **大量通用寄存器**：配备了大量的通用寄存器，以减少访存次数。
6.  **硬布线逻辑控制**：[[硬件-控制器]]采用[[技术-组合逻辑控制|组合逻辑（硬布线）]]实现，而非CISC中常用的[[技术-微程序控制|微程序控制]]，以提高执行速度。
7.  **重视编译优化**：依赖先进的优化编译器来高效地组合简单指令，完成复杂操作。

## RISC与CISC的比较

| 特性 | RISC (精简指令集) | CISC (复杂指令集) |
| :--- | :--- | :--- |
| **指令系统** | 数量少，功能简单 | 数量多，功能复杂 |
| **指令长度** | 固定 | 可变 |
| **访存操作** | 仅Load/Store指令 | 多种指令均可 |
| **寄存器** | 数量多，通用性强 | 数量少，常有专用寄存器 |
| **控制器** | 组合逻辑（硬布线） | 微程序控制 |
| **执行周期** | 大部分指令单周期完成 | 大部分指令多周期完成 |
| **编译** | 依赖优化编译器 | 编译器设计相对简单 |
