# 概念-机器数与真值

在计算机内部，所有数值都以二进制形式存储和运算。为了表示带有正负号的数，需要将符号“数字化”，这就引出了真值和机器数的概念。

- **真值 (True Value)**：指带有“+”或“-”符号的、符合人类习惯的数值。例如：+13, -5。
- **机器数 (Machine Number)**：指将符号数字化后的二进制数。通常规定最高位为符号位，**0表示正，1表示负**。

为了便于运算，特别是将减法转化为加法，有符号数主要有以下几种机器数表示法：

## 1. 原码 (Sign-Magnitude)
- **定义**：符号位加上真值的绝对值的二进制表示。
- **特点**：表示简单直观，但加减法运算复杂，且“0”有两种表示形式（+0 和 -0）。
- **示例 (8位整数)**：`[+13]原 = 0,0001101`；`[-13]原 = 1,0001101`。

## 2. 补码 (Two's Complement)
- **定义**：
    - **正数**：补码与原码相同。
    - **负数**：其原码除符号位外，各位取反，末位加1。
- **核心优势**：可以将减法运算转换为加法运算，`[A-B]补 = [A]补 + [-B]补`，从而简化[[硬件-运算器]]的设计。
- **特点**：“0”的表示唯一，比原码和反码能多表示一个负数。现代计算机普遍采用补码进行算术运算。
- **示例 (8位整数)**：`[+13]补 = 00001101`；`[-13]补 = 11110011`。

## 3. 反码 (One's Complement)
- **定义**：
    - **正数**：反码与原码相同。
    - **负数**：其原码除符号位外，各位取反。
- **特点**：主要作为原码和补码之间转换的中间过渡。“0”也有两种表示形式。
- **示例 (8位整数)**：`[+13]反 = 00001101`；`[-13]反 = 11110010`。

## 4. 移码 (Excess/Biased Representation)
- **定义**：在真值上加上一个固定的偏移量（偏置值），通常是 $2^n$。
- **特点**：移码的大小能直接反映其真值的大小，便于比较。常用于表示[[概念-定点数与浮点数|浮点数]]的阶码。
- **与补码关系**：移码与补码仅符号位相反。

> 把符号"数字化"的数称为机器数,而把带"+"或"-"符号的数称为真值。一旦符号数字化后,符号和数值就形成了一种新的编码。……这些编码就是原码、补码、反码和移码。
