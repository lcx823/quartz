# 技术-存储器校验

存储器校验是为了确保数据在存储和传输过程中的完整性和准确性而采用的技术。由于物理原因（如器件故障、电磁干扰），数据可能会发生错误（位翻转），校验码就是用来检测甚至纠正这些错误的。

## 1. 奇偶校验码 (Parity Check Code)

- **原理**：在原始数据位（如7位ASCII码）上附加一位**校验位**，使得整个码字中“1”的个数满足奇数（奇校验）或偶数（偶校验）的规则。
- **能力**：只能**检测出奇数位**的错误。如果同时有两位发生错误，奇偶性不变，则无法检测。它**不具备纠错能力**。
- **应用**：结构简单，常用于I/O设备的数据传输校验。

## 2. 汉明码 (Hamming Code)

- **原理**：一种具有**一位纠错能力**的编码。它通过在 `n` 位信息位中增加 `k` 位校验位，构成一个 `n+k` 位的码字。校验位的数量 `k` 必须满足公式：$2^k \ge n + k + 1$。
- **纠错过程**：
    1.  **编码**：根据预设的规则（如配偶原则），计算出 `k` 个校验位的值。
    2.  **检错**：接收端根据同样的规则重新计算一组新的检测位。
    3.  **定位**：如果新的检测位不全为0，则其组成的二进制数值就直接指出了发生错误的那一位的位置。
    4.  **纠错**：将出错位取反即可完成纠错。

> 汉明码是由Richard Hanming于1950年提出的,它具有一位纠错能力。……新增添的检测位数 k 应满足: $2^k \ge n + k + 1$

## 3. 循环冗余校验码 (CRC)

- **原理**：基于多项式除法（模2运算）来生成校验码。它能检测出多位错误和突发性错误。
- **能力**：检错能力极强，也能用于纠错，但实现比汉明码复杂。
- **应用**：广泛用于[[硬件-辅助存储器]]（如硬盘）和网络通信中，以确保大块数据的完整性。
