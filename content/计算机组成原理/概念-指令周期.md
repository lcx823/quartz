# 概念-指令周期

指令周期（Instruction Cycle）是[[硬件-CPU]]从[[硬件-主存储器]]取出一条指令并执行该指令所需的全部时间。它是CPU工作的基本单位，体现了计算机“取指-执行”的循环工作模式。

## 指令周期的基本阶段

一个基本的指令周期包含两个阶段：

1.  **取指周期 (Fetch Cycle)**：CPU从主存中取出指令，并对指令进行译码分析。
2.  **执行周期 (Execute Cycle)**：CPU执行指令所规定的操作。

## 扩展的指令周期

根据指令的不同，指令周期可能会包含额外的阶段：

- **间址周期 (Indirect Cycle)**：如果指令采用间接寻址，CPU需要在执行周期之前增加一个间址周期，用于从主存中读取操作数的有效地址。
- **中断周期 (Interrupt Cycle)**：如果在一条指令执行完毕后，CPU检测到有中断请求，它会进入中断周期。在此期间，CPU会保存当前程序的断点（PC内容），然后转向执行中断服务程序。

一个完整的指令周期流程可以概括为：**取指 -> 间址（可选） -> 执行 -> 中断（可选）**。

> CPU每取出并执行一条指令所需的全部时间称为指令周期,也即CPU完成一条指令的时间...一个完整的指令周期应包括取指、间址、执行和中断4个子周期。

## 各周期的数据流

- **取指周期**：`PC -> MAR -> M(MAR) -> MDR -> IR`, 同时 `(PC)+1 -> PC`。
- **间址周期**：`Ad(IR) -> MAR -> M(MAR) -> MDR -> Ad(IR)`。
- **执行周期**：数据流根据指令类型而异，可能涉及CPU内部寄存器、ALU和主存之间的数据传送。
- **中断周期**：`SP -> MAR`, `PC -> MDR -> M(MAR)`, `中断服务程序入口地址 -> PC`。
