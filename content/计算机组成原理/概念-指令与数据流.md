# 概念-指令与数据流

指令流和数据流是描述计算机执行程序时，指令和数据在系统中流动的抽象概念。它们是理解计算机工作过程和并行处理技术的基础。

## 1. 指令流 (Instruction Stream)

- **定义**：指在程序执行过程中，计算机从[[硬件-主存储器]]中按顺序取出的**指令序列**。
- **控制者**：指令流的流向主要由[[硬件-寄存器-PC|程序计数器（PC）]]控制。
    - **顺序执行**：PC自动加1，形成连续的指令流。
    - **转移/分支**：转移类指令会修改PC的值，改变指令流的走向。

## 2. 数据流 (Data Stream)

- **定义**：指在程序执行过程中，根据指令的要求，在[[硬件-CPU]]、[[硬件-主存储器]]和[[硬件-IO设备]]之间流动的**数据序列**。
- **驱动者**：数据流是由指令流驱动的。执行一条指令（如`LOAD`, `ADD`, `STORE`）会引发相应的数据在寄存器、ALU和存储器之间流动。

## 指令周期中的数据流

在不同的[[概念-指令周期]]阶段，指令和数据在CPU内部和外部总线上的流动路径是不同的：

- **取指周期的数据流**：核心是**指令**从主存流向CPU的[[硬件-寄存器-IR]]。
    - `(PC) -> MAR -> M(MAR) -> MDR -> IR`
- **间址周期的数据流**：核心是**有效地址**从主存流向CPU。
    - `Ad(IR) -> MAR -> M(MAR) -> MDR`
- **执行周期的数据流**：核心是**操作数**在主存、CPU寄存器和[[硬件-运算器|ALU]]之间流动。
- **中断周期的数据流**：核心是**程序断点**从[[硬件-寄存器-PC]]流向主存（通常是堆栈）。

理解指令流和数据流的分离与交互，是掌握[[技术-指令流水]]、数据相关性以及更高级并行计算架构（如SIMD, MIMD）的关键。
