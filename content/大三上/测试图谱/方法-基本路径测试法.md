# 方法-基本路径测试法

基本路径测试法（Basis Path Testing）是一种重要的[[./方法-白盒测试|方法-白盒测试]]技术，它通过分析程序的控制流程图，找出所有独立的、可执行的路径，并为每条路径设计测试用例。其目标是确保程序中的每条可执行语句至少被执行一次。

> 基本路径测试法是在程序控制流图的基础上，通过分析控制构造的环路复杂性，导出基本可执行路径集合，从而设计测试用例的方法。
> —— 第5章 5.3.3 基本路径测试法

## 设计步骤

1.  **绘制程序控制流程图**:
    - 将程序的代码逻辑转换为由节点（代表代码块）和边（代表控制流）组成的图形。

2.  **计算圈复杂度 (Cyclomatic Complexity)**:
    - 圈复杂度 `V(G)` 是一个衡量程序逻辑复杂度的度量，它也直接给出了程序中独立路径的数量，即基本路径集合的大小。
    - **计算方法**:
        - `V(G) = 区域数` (包括图外部区域)
        - `V(G) = 边数 - 节点数 + 2`
        - `V(G) = 判断节点数 + 1` (最常用)

3.  **确定基本路径**:
    - 根据控制流程图，找出 `V(G)` 条线性独立的路径。第一条路径通常选择最简单的、不经过循环的主路径，后续路径则在主路径基础上，每次只改变一个判断分支，直至覆盖所有边。

4.  **设计测试用例**:
    - 为每条基本路径设计一个或多个测试用例，确保这些路径在测试中能够被执行。测试用例的输入数据需要能驱动程序沿指定的路径运行。

- **计算圈复杂度**: 图中有3个判断节点(A, C, E)，所以 `V(G) = 3 + 1 = 4`。

- **设计测试用例**: 设计4个测试用例，分别驱动这4条路径的执行。

## 核心价值
- 它提供了一种系统性的方法来保证测试的覆盖度，确保所有代码逻辑都得到基本测试。
- 圈复杂度的计算本身也为评估代码的可维护性和可测试性提供了量化依据。
