# 方法-功能图法

功能图法是一种综合性的测试用例设计方法，它结合了**状态迁移图 (State Transition Diagram, STD)** 和**逻辑功能模型 (Logic Function Model, LFM)**，以同时描述程序的动态行为和静态逻辑。

> 功能图法就是为了解决动态说明问题的一种测试用例的设计方法。功能图由状态迁移图和逻辑功能模型构成。
> —— 第6章 6.2.6 功能图法

## 核心构成

1.  **状态迁移图 (STD)**:
    - **作用**: 描述系统的**动态行为**，即系统在不同输入下如何从一个状态迁移到另一个状态。
    - **表现形式**: 类似于程序的控制流程图，节点代表状态，弧线代表状态之间的迁移（由输入或事件触发）。
    - *示例*: 书中以系统登录功能的状态迁移为例（等待输入 -> 等待重新输入 -> 进入新状态）。

2.  **逻辑功能模型 (LFM)**:
    - **作用**: 描述系统的**静态逻辑**，即在某个特定状态下，输入条件和输出条件之间的对应关系。
    - **表现形式**: 通常使用[[./方法-因果图法|因果图]]或[[./方法-决策表方法|决策表]]来表示。
    - *示例*: 书中用决策表描述了登录时，不同用户名/密码组合对应的输出结果（成功/失败）和下一个状态。

## 测试用例设计思路

功能图法将测试用例的设计分为两个层次：
1.  **局部测试用例设计**:
    - **依据**: 逻辑功能模型（即决策表或因果图）。
    - **方法**: 采用[[./方法-黑盒测试|方法-黑盒测试]]技术（如决策表法），为**每个状态**设计测试用例，以覆盖该状态下所有可能的输入条件组合。
2.  **整体测试用例设计**:
    - **依据**: 状态迁移图。
    - **方法**: 采用[[./方法-白盒测试|方法-白盒测试]]技术（如路径覆盖），设计测试用例来覆盖状态迁移图中的**逻辑路径**，即从初始状态到最终状态的各种状态迁移序列。

## 本质
功能图法是一种综合运用[[./方法-白盒测试|方法-白盒测试]]和[[./方法-黑盒测试|方法-黑盒测试]]的混合方法。它用白盒的路径覆盖思想来测试宏观的状态迁移流程，同时用黑盒的逻辑组合思想来测试每个状态内部的具体功能。
