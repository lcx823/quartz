# 方法-条件覆盖

条件覆盖（Condition Coverage）是[[./方法-白盒测试|方法-白盒测试]]中一种关注判断语句内部逻辑的覆盖标准。它的目标是设计足够多的测试用例，使得程序中每个判断语句的**每个子条件**都至少取得一次“真”值和一次“假”值。

## 核心思想
条件覆盖不关心整个判断语句的最终结果，而是深入到构成判断的原子条件。例如，对于 `if (A and B)`，条件覆盖要求测试用例需要覆盖 `A=true`, `A=false`, `B=true`, `B=false` 这四种情况。

## 与其他覆盖标准的关系

- **不一定能保证分支覆盖**: 条件覆盖只关心每个子条件是否都取过真假值，不保证这些子条件的组合能够覆盖判断语句的所有分支。
    > 条件全部覆盖了，也不能覆盖全部分支。还是以判断 `{a > 0 and b > 0}` 为例，`a = 1, b = -1` 和 `a = -1, b = 1` 覆盖了4个条件 `a>0`、`a<=0`、`b>0`、`b<=0`，但结果只覆盖了分支二（F.），而没有覆盖分支一。
    > —— 第5章 5.3.2 条件覆盖法
- **强于语句覆盖，但与分支覆盖无必然包含关系**:
    - **条件-分支覆盖 (Condition-Decision Coverage)**: 为了弥补各自的不足，实践中常将两者结合，要求既满足分支覆盖，又满足条件覆盖。
    - **组合覆盖 (Multiple Condition Coverage)**: 更严格的标准，要求覆盖判断语句中所有子条件的所有可能组合。对于 `n` 个子条件，需要 `2^n` 个测试用例，成本较高。

## 示例
考虑代码 `if (a > 0 and b > 0)`，子条件为 `a>0` 和 `b>0`。
- **要满足条件覆盖**: 需要的测试用例必须覆盖以下4种情况：
    1.  `a > 0` (真)
    2.  `a <= 0` (假)
    3.  `b > 0` (真)
    4.  `b <= 0` (假)
- **可能的测试用例集**:
    - 用例1: `a=1, b=1` (覆盖 `a>0` 和 `b>0`)
    - 用例2: `a=-1, b=-1` (覆盖 `a<=0` 和 `b<=0`)
- 这个测试集满足了条件覆盖，但只测试了`if`为`true`和`if`为`false`两种情况，恰好也满足了分支覆盖。但如果选择的测试集是 `a=1, b=-1` 和 `a=-1, b=1`，则只满足条件覆盖，不满足分支覆盖。
