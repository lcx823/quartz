![[./assets/8、自动化测试/c8.png|c8.png]]
## 8.1 自动化测试概述

### 8.1.1 自动化测试的概念与演变
**定义**：自动化测试（automated test）是相对手工测试（manual test）的概念，指利用测试工具自动执行测试用例的操作过程，将人工驱动的测试行为转化为机器执行。
**演变历程**：
1. **萌芽期**：出于“偷懒”想法，代码替代简单重复工作。
2. **发展期**：商业工具为代表，经历**线性测试** -> **模块化驱动测试** -> **数据驱动测试** -> **关键字驱动**四个阶段。
3. **测试爆发期**：结合Devops、敏捷、Docker等新技术，开源工具与框架层出不穷，**CI/CD闭环**成为趋势。
4. **未来**：AI-测试终结者。

### 8.1.2 自动化测试适用条件
1. **项目需求变动不频繁**：需求变动会导致脚本维护成本增加，变动过频则不适合。
2. **项目进度压力不大，时间不紧迫**：设计框架、编写调试脚本需要足够时间，匆忙实施难以保证质量。
3. **多种浏览器或平台上可以重复运行**：脚本复用率高才能体现经济价值，抵消编写成本。

### 8.1.3 自动化测试金字塔策略
在敏捷开发中，测试策略通常呈金字塔状：
1. **单元测试 (底层)**：针对功能模块（函数、类方法）。主要用**白盒测试**，关注内部逻辑。
2. **接口测试 (中间层)**：针对数据传输、数据库性能。主要用**黑盒与白盒结合**，保证数据完整性，对扩展维护重要。
3. **UI测试 (顶层)**：关注用户体验。并非完全自动化，需人工辅助确认易用性。

### 8.1.4 手工测试 VS 自动化测试

| 特性 | 手工测试 | 自动化测试 |
| :--- | :--- | :--- |
| **效率/速度** | 低 | **高** |
| **复用性** | 低 | **高** |
| **覆盖率度量** | 困难 | **容易** |
| **准确/可靠性** | 低（易疲劳） | **高**（不知疲劳） |
| **发现缺陷率** | **高** | 低（机械） |
| **一次性投入** | 低 | **大** |
| **灵活性** | **高**（创造性） | 低 |

**优缺点总结**：
*   **优点**：
    *   **提高回归测试效率**：重复执行相同轨迹。
    *   **提高测试精确度**：机器执行更精准。
    *   **提高人员利用率**：无人值守，释放人力。
    *   **提高测试便捷性**：易于实现负载/压力测试。
*   **缺点**：
    *   **不能提高有效性**：脚本可能存在逻辑错误。
    *   **发现缺陷能力有限**：无主观判断，主要用于缺陷预防和回归，不易发现新缺陷。

## 8.2 自动化测试常见技术

### 1. 录制与回放技术
*   **原理**：测试人员操作一遍，工具记录通信过程和交互行为生成脚本。
*   **应用**：回放以检查错误（如图片加载、URL访问）。

### 2. 脚本技术
*   **线性脚本**：录制得到，包含鼠标点击、页面选择等，可完整回放。
*   **结构化脚本**：类似结构化程序设计，包含顺序、分支、循环逻辑及函数调用，灵活测试复杂功能。
*   **共享脚本**：一个脚本可被其他脚本调用，实现复用。

### 3. 数据驱动技术
*   **原理**：数据与脚本分离。从数据文件（CSV, Excel, Database等）读取输入数据作为参数。
*   **优点**：提高脚本复用率和可维护性。
*   **局限**：不适合界面变化较大的项目。

### 4. 关键字驱动
*   **原理**：数据驱动的改进。将**数据与脚本分离**、**界面元素与内部对象分离**、**测试过程与实现细节分离**。
*   **核心**：通过关键字（被操作对象、操作、值）描述测试逻辑。

### 5. 行为驱动 (BDD)
*   **原理**：基于业务需求和场景设计用例。需开发、测试、业务分析人员协作。
*   **关注点**：软件内部运作变化，解决单元测试细节问题。

## 8.3 搭建自动化测试环境

### 8.3.1 安装 Python 和 PyCharm
*   下载并安装 Python 解释器。
*   安装 PyCharm 编辑器。

### 8.3.2 安装 Selenium
**Selenium** 是用于Web应用程序测试的工具，支持多浏览器、多系统、多语言。通过 **WebDriver** 直接与浏览器交互。
*   **安装方式**：
    1.  `pip install selenium`
    2.  通过 PyCharm 安装。

### 8.3.3 安装浏览器驱动
**作用**：将测试脚本转换为浏览器指令。
*   **对应关系**：
    *   Chrome -> `chromedriver.exe`
    *   Firefox -> `geckodriver.exe`
    *   IE -> `IEDriverServer.exe`
*   **注意**：驱动版本需与浏览器大版本号一致（如Chrome 100对应驱动100）。

## 8.4 自动化测试框架

### 8.4.1 常用框架
*   **unittest**：Python标准库自带。
*   **pytest**：第三方框架，简洁高效，兼容unittest。
*   **Robot Framework**。

### 8.4.2 unittest 框架
**特点**：无需安装，丰富断言，可生成报告。
**核心要素**：
1.  **TestCase**：测试用例基类。测试方法必须以 `test` 开头。
2.  **TestSuite**：测试套件，容器，用来装载多个TestCase。使用 `addTest()` 添加。
3.  **TextTestRunner**：测试执行器，运行测试并返回结果。
4.  **TextTestResult**：保存测试结果。
5.  **Fixture**：测试固件。`setUp()` 初始化环境，`tearDown()` 清理环境。

**常用断言**：
*   `assertTrue(expr)`
*   `assertEqual(first, second)`
*   `assertIn(member, container)`
*   `assertIsNone(obj)`

### 8.4.3 pytest 框架
**特点**：第三方库（需 `pip install pytest`），简洁高效，支持丰富的插件（如Selenium, Appium, requests）。
**配置**：在 PyCharm 中需设置 Default test runner 为 pytest。
**命名规则**：
*   测试类以 `Test` 开头。
*   测试方法/函数以 `test` 开头。

**断言**：直接使用 Python 的 `assert` 关键字。
*   `assert a == b`
*   `assert a != b`
*   `assert a` (判断真)
*   `assert a in b`

## 8.5 实例：学成在线教育平台测试

### 8.5.1 测试环境
*   **项目**：学成在线教育平台。
*   **工具**：Python 3.8+, PyCharm, Katalon Recorder, Selenium, Chrome & ChromeDriver。

### 8.5.2 元素定位
1.  **浏览器开发者工具 (F12)**：使用选择器定位元素，查看 id, class, name 等属性。
2.  **Katalon Recorder 插件**：
    *   **下载安装**：浏览器扩展程序。
    *   **录制**：创建 Test Suite -> Record -> 操作页面 -> Stop。
    *   **导出**：Export -> 选择语言（如 Python WebDriver） -> 查看/复制代码。
    *   **作用**：辅助脚本编写，不直接用于生产代码（未封装易出错）。

### 8.5.3 测试脚本编写
**场景**：登录 -> 退出。
**代码逻辑**（基于 unittest）：
1.  导入 `selenium`, `unittest`。
2.  定义测试类继承 `unittest.TestCase`。
3.  `setUp`：启动浏览器，最大化，隐式等待。
4.  测试方法 `test_login_and_logout`：
    *   打开 URL。
    *   定位并点击“登录”。
    *   输入账号、密码。
    *   点击登录按钮。
    *   断言登录成功（如检查是否存在“退出”按钮）。
    *   鼠标悬停/点击“退出”。
    *   断言退出成功。
5.  `tearDown`：关闭浏览器 `driver.quit()`。

### 8.5.4 页面跳转测试
**场景**：首页 -> 注册页 / 课程页。
**关键点**：
*   窗口切换（handle）。
*   定位文本链接（link_text）。
