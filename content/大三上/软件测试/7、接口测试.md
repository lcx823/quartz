# 第七章 接口测试

![[./assets/7、接口测试/c7.png|c7.png]]
## 7.1 接口测试简介

### 7.1.1 接口与接口测试
- **接口 (Interface)**：指系统或组件之间进行信息交互的通道。随着软件功能日益复杂，接口数量不断增加，确保接口正常交互至关重要。
- **接口测试 (Interface Testing)**：对系统组件间接口进行的测试。
    - **原理**：模拟客户端向服务器发送请求，服务器处理后返回响应数据。
    - **关注点**：
        - 接口间数据的传递、交换和控制管理过程。
        - 接口间的逻辑依赖关系。
        - 服务器返回的响应数据是否符合预期。

### 7.1.2 接口测试的实现方式

#### 1. 通过工具实现
- **常用工具**：
    - **Postman**：简单易用，支持批量运行、保存历史记录，适合编程基础较弱的人员。
    - **JMeter**：Apache开源工具，支持多协议，功能强大，可进行性能测试和接口测试。
- **优势**：提高测试效率，易于学习掌握。

#### 2. 通过代码实现
- **常用语言与库**：
    - **Java**：结合 `HttpClient` 技术。
    - **Python**：结合 `Requests` 框架。
- **优势**：与开发技术栈统一，更好地对接测试流程。
- **劣势**：对测试人员编程能力要求较高。

### 7.1.3 接口测试流程
通常包括以下步骤：
1.  分析接口测试需求。
2.  解析与评审接口文档（API文档）。
    - 接口文档描述接口信息，若缺失可通过抓包工具获取。
3.  编写接口测试计划。
4.  设计与评审接口测试用例。
    - 用例包含：ID、名称、接口名称、前置条件、URL、请求方法、请求头、参数、预期结果等。
5.  搭建接口测试环境。
6.  编写接口测试脚本。
7.  执行接口测试用例。
8.  管理与跟踪接口缺陷。
9.  整理测试报告。

## 7.2 HTTP 协议基础

### 7.2.1 统一资源定位符 (URL)
URL (Uniform Resource Locator) 是互联网上标准的资源地址。
- **语法格式**：
  $$
  protocol://hostname:port/path?parameters?query\&fragment
  $$
- **组成部分**：
    - `protocol`：协议（如 HTTP, HTTPS, FTP）。
    - `hostname`：主机名或IP地址。
    - `port`：端口号（HTTP默认80，HTTPS默认443）。
    - `path`：资源路径，用 `/` 分隔。
    - `parameters`：查询参数，用 `?` 与路径分隔，多个参数用 `&` 连接。
    - `query`：查询字符串。
    - `fragment`：信息片段（锚点）。

> **示例**：`http://www.example.com:80/index.html?id=001&page=1`

### 7.2.2 HTTP 请求
HTTP 请求由客户端发送给服务器，主要包含三部分：
1.  **请求行 (Request Line)**：
    - 包含请求方法（GET, POST, PUT, DELETE等）、请求URL、协议版本。
2.  **请求头 (Request Headers)**：
    - 包含客户端环境信息、身份验证信息等。
    - 常见字段：`Host`, `User-Agent`, `Accept`, `Content-Type`。
3.  **请求体 (Request Body)**：
    - 客户端发送给服务器的数据，通常用于 POST/PUT 请求。
    - 常见格式：`text/html`, `application/json`, `multipart/form-data`。

### 7.2.3 HTTP 响应
HTTP 响应由服务器返回给客户端，主要包含三部分：
1.  **状态行 (Status Line)**：
    - 包含协议版本、**状态码**、状态描述。
    - **常见状态码**：
        - `200 OK`：请求成功。
        - `400 Bad Request`：客户端请求语法错误。
        - `401 Unauthorized`：未授权。
        - `403 Forbidden`：拒绝服务。
        - `404 Not Found`：资源不存在。
        - `500 Internal Server Error`：服务器内部错误。
        - `503 Server Unavailable`：服务不可用。
2.  **响应头 (Response Headers)**：
    - 包含服务器信息、内容类型等。
    - 常见字段：`Server`, `Content-Type`, `Content-Length`。
3.  **响应体 (Response Body)**：
    - 服务器返回的实际数据（如 HTML, JSON）。

## 7.3 Postman 入门

### 7.3.1 安装与界面
- 注册登录后进入主界面，主要操作区域包括左侧的集合（Collections）管理和右侧的请求编辑器。

### 7.3.2 发送 HTTP 请求步骤
1.  **创建集合 (Collection)**：用于组织管理请求。
2.  **添加请求 (Add Request)**：
    - 设置请求方法（如 GET）。
    - 输入请求 URL。
    - 设置参数（Params）、授权（Authorization）、请求头（Headers）、请求体（Body）。
3.  **发送请求 (Send)**：点击 Send 按钮。
4.  **查看响应**：在下方查看响应体、状态码、响应时间等。

## 7.4 Postman 的基本使用

### 7.4.1 断言 (Tests)
断言用于验证实际结果与预期结果是否一致。Postman 使用 JavaScript 编写断言。
- **常用断言类型**：
    1.  **响应状态码断言**：验证状态码是否为 200。
        ```javascript
        pm.test("Status code is 200", function () {
            pm.response.to.have.status(200);
        });
        ```
    2.  **包含指定字符串断言**：验证响应体包含特定文本。
        ```javascript
        pm.test("Body matches string", function () {
            pm.expect(pm.response.text()).to.include("string_you_want_to_search");
        });
        ```
    3.  **JSON 数据断言**：验证 JSON 响应中的具体字段值。
        ```javascript
        pm.test("Your test name", function () {
            var jsonData = pm.response.json();
            pm.expect(jsonData.value).to.eql(100);
        });
        ```

### 7.4.2 接口关联
当一个接口的输入依赖于另一个接口的输出时，需要进行接口关联。
- **实现步骤**：
    1.  **提取数据**：在第一个接口的 `Tests` 中提取响应数据。
        ```javascript
        var jsonData = pm.response.json();
        // 设置全局变量
        pm.globals.set("variable_key", jsonData.value);
        // 或设置环境变量
        pm.environment.set("variable_key", jsonData.value);
        ```
    2.  **引用变量**：在第二个接口的参数中使用 `{{variable_key}}` 引用。

### 7.4.3 参数化
通过数据文件实现测试数据的参数化，支持批量测试。
- **支持文件格式**：CSV, JSON。
- **步骤**：
    1.  准备数据文件（如 csv，第一行为变量名，后续为数据）。
    2.  在请求中使用 `{{variable_name}}` 引用变量。
    3.  使用 **Collection Runner** 运行集合。
    4.  在 Runner 中选择数据文件 (Select File)。
    5.  设置迭代次数 (Iterations) 对应数据行数。
    6.  运行并查看结果。

### 7.4.4 生成测试报告
Postman 本身不直接生成详细报告，需借助 **Newman** 工具。
- **安装环境**：
    1.  安装 **Node.js**。
    2.  安装 **Newman**：`npm install -g newman`
    3.  安装 HTML 报告插件：`npm install -g newman-reporter-html`
- **生成步骤**：
    1.  导出 Postman 集合 (Collection) 和环境变量 (Environment) 为 JSON 文件。
    2.  在命令行执行 Newman 命令：
        $$
        newman \ run \ <collection.json> \ -e \ <environment.json> \ -r \ html
        $$
    3.  在目录下查看生成的 HTML 报告。
