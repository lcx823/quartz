# 架构-统一内存访问 (UMA)

**统一内存访问 (Uniform Memory Access, UMA)** 是一种共享存储器的多处理机系统结构。在这种结构中，所有CPU共享一个物理内存，并且任何CPU访问内存中任何位置所需的时间都是**相同**的。UMA结构通常用于[[概念-多处理机系统|对称多处理机(SMP)]]系统中。

## 1. 使用单总线的SMP结构
- **结构**: 多个CPU通过一条**公用总线**连接到一个集中的共享内存。
- **优点**: 结构简单，为单处理机编写的程序易于移植。
- **缺点**: **总线成为瓶颈**。随着CPU数量的增加，对总线的争用加剧，限制了系统的可伸缩性。
- **改进**: 为每个CPU配置一个**高速缓存 (Cache)**，可以显著减少对公用总线的访问频率。

## 2. 使用多总线的SMP结构
- **结构**: 每个CPU除了通过系统总线访问共享内存外，还拥有自己的**本地私有内存**，通过本地总线访问。
- **优点**: 将只读代码、私有数据等放在本地内存，大大减少了对系统总线的争用，支持更多的CPU。
- **缺点**: 对编译器和程序员要求更高，需要仔细安排数据存放位置。

## 3. 使用单级交叉开关的SMP结构
- **结构**: 利用**交叉开关 (Crossbar Switch)** 阵列将CPU和存储器模块连接起来。每个CPU到每个存储模块都有一条专用的通路。
- **优点**: 避免了总线争用，实现了真正的并行访存。
- **缺点**: 交叉开关的成本与端口数的平方 ($N^2$) 成正比，硬件成本高昂，限制了系统规模。

## 4. 使用多级交换网络的SMP结构
- **结构**: 将多个小的交叉开关分级连接起来，构成一个多级交换网络。
- **优点**: 在CPU和存储模块之间提供了多条路径，减少了阻塞概率，比单级交叉开关成本低。
- **缺点**: 硬件依然昂贵且复杂。
