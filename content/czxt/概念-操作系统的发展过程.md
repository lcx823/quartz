# 概念-操作系统的发展过程

操作系统的发展历程反映了计算机技术的演进和用户需求的不断变化，主要经历了以下几个阶段：

1.  **未配置操作系统的计算机系统 (人工操作方式)**
    - **时期**: 1945年 - 20世纪50年代中期
    - **特点**: 用户通过纸带或卡片输入程序和数据，独占全机，CPU等待人工操作，效率低下。
    - **改进**: 引入**脱机I/O方式**，通过外围机将数据输入磁带，再由主机高速调入内存，减少了CPU的空闲时间。

2.  **[[架构-单道批处理系统]]**
    - **时期**: 20世纪50年代中期
    - **特点**: 作业成批处理，内存中始终只有一道作业。通过监督程序实现作业的自动接续，但资源利用率仍然不高。

3.  **[[架构-多道批处理系统]]**
    - **时期**: 20世纪60年代中期
    - **核心技术**: **多道程序设计**，内存中同时存放若干道作业，它们[[概念-操作系统的基本特性|并发]]执行。
    - **优点**: 资源利用率高，系统吞吐量大。
    - **缺点**: 平均周转时间长，无交互能力。

4.  **[[架构-分时系统]]**
    - **发展动力**: 满足用户对人机交互的需求。
    - **核心技术**: 时间片轮转，允许多个用户通过终端同时以交互方式使用计算机。
    - **特征**: 多路性、独立性、及时性、交互性。

5.  **[[架构-实时系统]]**
    - **发展动力**: 满足工业控制、武器控制等领域的实时响应需求。
    - **核心特征**: 将时间作为关键参数，必须在规定时间内完成对外部事件的处理。
    - **任务类型**:
        - **硬实时(HRT)任务**: 必须严格满足截止时间。
        - **软实时(SRT)任务**: 偶尔错过截止时间影响不大。

6.  **微机操作系统**
    - **发展**: 随着VLSI技术发展而出现，经历了从8位到64位的演进。
    - **分类**:
        - **单用户单任务OS**: 如CP/M, MS-DOS。
        - **单用户多任务OS**: 如Windows 95/98。
        - **多用户多任务OS**: 如UNIX, Linux, Windows NT系列。

7.  **嵌入式操作系统**
    - **特点**: 应用于嵌入式系统，系统内核小、系统精简、实时性高、可配置性强。
    - **实例**: `μC/OS-II`, 嵌入式Linux, VxWorks, Android, iOS。

8.  **网络操作系统 (NOS)**
    - **功能**: 在计算机网络环境下管理网络资源，实现数据通信和资源共享。
    - **特征**: 硬件独立性、接口一致性、资源透明性、系统可靠性、执行并行性。

9.  **分布式操作系统**
    - **定义**: 配置在分布式系统上的公用OS，以全局方式对所有资源进行统一管理。
    - **特征**: 分布性、透明性、同一性、全局性。
    - **实例**: 华为鸿蒙系统 (HarmonyOS)。
