# 架构-非统一内存访问 (NUMA)

**非统一内存访问 (Non-Uniform Memory Access, NUMA)** 是一种为了解决[[架构-统一内存访问 (UMA)|UMA]]结构可伸缩性有限问题而设计的多处理机系统结构。它允许构建包含数十个甚至数百个CPU的大型系统。

## 1. NUMA结构与特点
- **结构**:
  - 系统由多个**节点 (Node)** 组成，节点间通过高速互连模块（如总线）连接。
  - 每个节点包含若干个CPU和一个**本地内存 (Local Memory)**。
  - 所有节点的本地内存共同构成了系统的**全局地址空间**，任何CPU都可以访问。
- **核心特点**: **访问延迟不一致**。
  - CPU访问其所在节点的**本地内存**速度最快。
  - CPU访问其他节点的**远程内存 (Remote Memory)** 速度较慢，因为需要通过互连模块。

## 2. CC-NUMA (Cache Coherent NUMA)
- **概念**: 为了减少对远程内存的访问，NUMA系统通常会为每个CPU配备专属的**高速缓存**，并需要机制来保证所有CPU缓存中的数据副本的一致性，这种结构称为**缓存一致性NUMA (CC-NUMA)**。
- **实现方法 - 基于目录的多处理机**:
  - **基本思想**: 不再使用监听总线的方式来维护缓存一致性，而是为每个内存块维护一个**目录 (Directory)** 项。
  - **目录内容**: 该目录记录了此内存块被哪些节点的CPU缓存了，以及缓存的状态（独占/修改、共享/只读）。
  - **工作流程**:
    1.  当一个CPU要读写某个内存块时，它会向该内存块所在的“宿主”节点发送请求。
    2.  宿主节点查询该内存块的目录项。
    3.  根据目录信息，宿主节点协调其他缓存了该块的CPU，使它们的缓存副本失效或更新，从而保证数据一致性。
- **优点**: 解决了UMA架构的可伸缩性问题，能够支持大规模的多处理机系统。
- **缺点**: 访问远程内存的延迟仍然是一个性能瓶颈，对程序员和OS调度器提出了更高的要求，需要尽量将进程及其所需数据放置在同一个节点内。
