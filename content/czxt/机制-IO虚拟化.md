# 机制-I/O虚拟化

I/O虚拟化负责管理[[概念-虚拟化|虚拟机]]与共享物理I/O设备之间的I/O请求路由。

## 1. 全设备模拟 (Full Device Emulation)
- **原理**: [[机制-虚拟机监视器 (VMM)|VMM]]通过纯软件的方式，为虚拟机模拟出一个完整的、与真实硬件型号无关的虚拟设备（如虚拟网卡、虚拟磁盘控制器）。
- **工作流程**: 客户机OS的设备驱动程序与这个虚拟设备交互，所有I/O请求都会陷入到VMM中，由VMM模拟硬件行为并转发给真实的物理设备。
- **优点**: **兼容性好**，无需修改客户机OS，可以运行任何操作系统。
- **缺点**: **性能差**，因为每次I/O操作都需要VMM的软件介入和模拟，路径长，开销大。

## 2. 半虚拟化I/O (Paravirtualized I/O)
- **原理**: 采用**分离式驱动模型 (Split Driver Model)**，需要客户机OS安装专门的驱动程序。
- **结构**:
  - **前端驱动 (Frontend Driver)**: 运行在客户机OS中，负责收集I/O请求。
  - **后端驱动 (Backend Driver)**: 运行在宿主域（Host OS或Domain 0）中，负责与物理设备交互。
  - **通信**: 前端和后端驱动通过高效的共享内存和事件通道进行通信。
- **优点**: **性能较高**，因为它绕过了VMM的模拟层，提供了一个更优化的I/O路径。
- **缺点**: **需要修改客户机OS**来安装前端驱动，降低了兼容性。

## 3. 直接I/O (Direct I/O) / I/O Passthrough
- **原理**: 允许虚拟机**直接访问和控制**物理I/O设备，绕过了VMM。
- **硬件支持**: 需要CPU和主板芯片组的硬件支持，如**Intel VT-d**或**AMD-Vi (IOMMU)**。
  - **IOMMU (I/O Memory Management Unit)**: 负责将设备的DMA地址（虚拟机物理地址）转换为真实的主机物理地址，并提供设备间的隔离，保证安全性。
- **优点**: **性能接近原生**，因为I/O路径最短，VMM介入最少。
- **缺点**: 一个物理设备在同一时间只能分配给一个虚拟机，**无法实现设备共享**。

> 现代虚拟化解决方案通常会**混合使用**这些技术。例如，对性能要求不高的设备（如鼠标）使用全设备模拟，而对性能关键的设备（如网卡、磁盘）使用半虚拟化或直接I/O。
