# 第15章 云借阅图书管理系统

## 15.1 系统概述

### 15.1.1 系统功能介绍
云借阅图书管理系统后台使用 **SSM (Spring + Spring MVC + MyBatis)** 框架编写，前台页面使用 **Bootstrap** 和 **jQuery** 框架编写。

系统主要包含两大功能模块：
1.  **用户登录模块**：实现用户的登录与注销。
2.  **图书管理模块**：
    *   **新书推荐**：查询新上架图书、借阅图书。
    *   **图书借阅**：查询图书、新增图书、编辑图书、借阅图书。
    *   **当前借阅**：查询当前借阅、归还图书、确认归还。
    *   **借阅记录**：新增借阅记录、查询借阅记录。

### 15.1.2 系统架构设计
系统采用典型的 Java EE 三层架构：
1.  **持久对象层 (POJO)**：由若干持久化类（实体类）组成。
2.  **数据访问层 (DAO层)**：由 DAO 接口和 MyBatis 映射文件组成。DAO 接口统一以 `Mapper` 结尾。
3.  **业务逻辑层 (Service层)**：由 Service 接口和实现类组成，实现业务逻辑。
4.  **Web 表现层**：包括 Spring MVC 的 Controller 类和 JSP 页面。

### 15.1.3 文件组织结构
项目遵循 Maven 的标准目录结构，并在 `src/main/java` 下通过包名区分不同的层（如 `controller`, `service`, `dao`, `domain` 等）。

### 15.1.4 系统开发及运行环境
*   **操作系统**：Windows 7
*   **Web服务器**：Tomcat 8.5.24
*   **Java开发包**：JDK 8
*   **开发工具**：IntelliJ IDEA 2019.3.2
*   **数据库**：MySQL 5.7.17
*   **浏览器**：Mozilla Firefox 84.0 (64位)

## 15.2 数据库设计

系统涉及的主要数据表如下：

1.  **用户表 (`user`)**
    *   `user_id` (主键): 用户ID
    *   `user_name`: 用户名称
    *   `user_password`: 用户密码
    *   `user_email`: 用户邮箱（账号）
    *   `user_role`: 用户角色 (ADMIN: 管理员, USER: 普通用户)
    *   `user_status`: 用户状态 (0: 正常, 1: 禁用)

2.  **图书信息表 (`book`)**
    *   `book_id` (主键): 图书编号
    *   `book_name`: 图书名称
    *   `book_isbn`: ISBN编号
    *   `book_press`: 出版社
    *   `book_author`: 作者
    *   `book_pagination`: 页数
    *   `book_price`: 价格
    *   `book_uploadtime`: 上架时间
    *   `book_status`: 状态 (0: 可借阅, 1: 已借阅, 2: 归还中, 3: 已下架)
    *   `book_borrower`: 借阅人
    *   `book_borrowtime`: 借阅时间
    *   `book_returntime`: 预计归还时间

3.  **借阅记录表 (`record`)**
    *   `record_id` (主键): 记录ID
    *   `record_bookname`: 图书名称
    *   `record_bookisbn`: ISBN编号
    *   `record_borrower`: 借阅人
    *   `record_borrowtime`: 借阅时间
    *   `record_remandtime`: 归还时间

## 15.3 系统环境搭建

在 `pom.xml` 中引入以下依赖：
*   **Spring框架**：`spring-context`, `spring-tx`, `spring-jdbc`, `spring-webmvc`
*   **MyBatis框架**：`mybatis`

### 15.3.2 准备数据库资源
创建数据库 `cloudlibrary` 并导入 `cloudlibrary.sql` 脚本。

### 15.3.3 准备项目环境
1.  **创建项目**：创建 Maven Web 项目 `cloudlibrary`。
2.  **配置文件**：

## 15.4 用户登录模块

### 15.4.1 用户登录
1.  **POJO类**：创建 `User` 类。
2.  **DAO层**：创建 `UserMapper` 接口及 XML，编写 `login(User user)` 方法。
3.  **Service层**：创建 `UserService` 接口及实现类，调用 DAO 层方法。
4.  **Controller层**：创建 `UserController`，处理登录请求 `/login`。将用户信息存入 Session (`USER_SESSION`)。

### 15.4.2 实现登录验证 (拦截器)
为了防止未登录用户直接访问后台页面，使用 **拦截器** 进行权限控制。

1.  **创建拦截器**：实现 `HandlerInterceptor` 接口（或继承 `HandlerInterceptorAdapter`）。
    *   在 `preHandle` 方法中判断 Session 中是否存在 `USER_SESSION`。
    *   如果存在或请求的是登录相关 URL，放行 (`return true`)。
    *   否则，转发到登录页面，返回 `false`。
2.  **配置拦截器**：在 `spring-mvc.xml` 中配置 `<mvc:interceptors>`。

### 15.4.3 注销登录
1.  **Controller层**：在 `UserController` 中添加 `logout` 方法。
2.  **实现**：调用 `session.invalidate()` 清除 Session，然后重定向到登录页面。

## 15.5 图书管理模块

### 15.5.1 新书推荐
展示最新上架的图书。
1.  **DAO层**：编写 `selectNewBooks()` 方法，按上架时间倒序查询图书。
2.  **Service层**：调用 DAO 层，并使用 **PageHelper** 进行分页处理。
3.  **Controller层**：处理 `/book/selectNewbooks` 请求，返回分页结果 `PageResult`。
4.  **页面**：使用 JSTL 遍历展示图书列表。

### 15.5.2 图书借阅
包含查询、新增、编辑、借阅功能。
1.  **查询图书**：支持按名称、作者、出版社模糊查询。
    *   DAO: 动态 SQL 拼接查询条件 `<if test="...">`。
2.  **新增图书**：
    *   DAO: `addBook(Book book)`。
    *   Service: 设置上架时间为当前时间，调用 DAO。
3.  **编辑图书**：
    *   DAO: `editBook(Book book)`。
    *   Controller: 先查询图书回显，再提交修改。
4.  **借阅图书**：
    *   实际上是**更新图书状态**。
    *   将图书状态设为 "1" (已借阅)，设置借阅人、借阅时间、预计归还时间。

### 15.5.3 当前借阅
展示当前用户借阅未归还的图书（普通用户）或所有借阅未归还的图书（管理员）。
1.  **DAO层**：
    *   `selectMyBorrowed()`: 查询当前用户借阅中图书。
    *   `selectBorrowed()`: 查询所有借阅中图书。
2.  **归还图书**：
    *   用户发起归还申请，将图书状态设为 "2" (归还中)。
3.  **确认归还**：
    *   管理员确认，将图书状态设为 "0" (可借阅)，清空借阅人信息。

### 15.5.4 借阅记录
记录每一次完整的借阅流程（借阅 -> 归还）。
1.  **新增记录**：在管理员**确认归还**时触发。
    *   调用 `RecordService.addRecord()`，将借阅信息写入 `record` 表。
2.  **查询记录**：支持按借阅人、图书名称查询借阅历史。

## 15.6 访问权限控制

为了防止普通用户通过 URL 直接访问管理员功能（如新增、编辑图书），需要进行权限控制。

**实现思路**：
1.  **定义普通用户权限**：创建配置文件 `ignoreUrl.properties`，列出普通用户允许访问的 URL（如查询、借阅）。
2.  **拦截器控制**：在拦截器中读取该配置文件。
3.  **逻辑判断**：
    *   如果是管理员 (`USER_SESSION.role == 'ADMIN'`)，放行所有请求。
    *   如果是普通用户，判断请求 URL 是否在允许列表中。如果不在，拦截并跳转到无权限提示或登录页。
