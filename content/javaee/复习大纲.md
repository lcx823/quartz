### **第一部分：MyBatis 核心 (第2章)**

#### **1. MyBatis 概述与核心概念**

*   **MyBatis 定义 (P3)**: 需掌握其基本定义——MyBatis 是一个支持自定义SQL、存储过程和高级映射的**持久层框架**。它通过简单的XML或注解来配置和映射，将接口与POJO（普通Java对象）映射成数据库中的记录。
*   **ORM 概念 (P5)**: **必须理解什么是ORM（对象关系映射）**，即对象模型和关系型数据库之间的映射。
*   **三大核心对象 (P11)**: **重点掌握**，三者之间存在明确的创建关系。
    1.  `SqlSessionFactoryBuilder`:
        *   **作用**: 读取配置文件，构建 `SqlSessionFactory`。生命周期仅限于方法内部，一旦工厂创建完成即可丢弃。
        *   **创建方式 (P12)**: 通过 `new SqlSessionFactoryBuilder().build(inputStream)` 的方式读取配置流来创建工厂。
    2.  `SqlSessionFactory`:
        *   **作用**: 创建 `SqlSession` 对象，是“工厂的工厂”。
        *   **线程安全性**: **线程安全**，适合**单例模式**。一旦创建，应在应用运行期间一直存在，无需重复创建。
        *   **核心方法 (P16)**: 通过调用 `openSession()` 方法来获取 `SqlSession`。
    3.  `SqlSession`:
        *   **作用**: 操作数据库的核心接口，用于执行SQL命令、获取映射器和管理事务。
        *   **线程安全性**: **线程不安全**，绝不能在多线程环境下共享，必须在每个线程中独立创建和销毁。适合**原型模式**。
        *   **核心方法 (P14)**:
            *   查询: `selectOne()` (查单个), `selectList()` (查多个)。
            *   增删改: `insert()`, `update()`, `delete()`，返回值均为 `int` 类型（影响的行数）。
            *   事务控制: `commit()` (提交), `rollback()` (回滚), `close()` (关闭)。
        *   **标准使用模板 (P14)**: 掌握在 `try-catch-finally` 结构中正确使用 `commit`, `rollback`, `close` 的方法。
            *   `try` 块中执行 `commit()`。
            *   `catch` 块中执行 `rollback()`。
            *   `finally` 块中执行 `close()`。

#### **2. MyBatis 工作原理 (P9-10)**

*   虽然无需背诵，但**必须能结合原理图（P9）清晰描述其工作流程**：
    1.  **准备阶段**: 准备核心配置文件 (`mybatis-config.xml`) 和 Mapper 映射文件 (`.xml`)。
    2.  **读取与构建**: `SqlSessionFactoryBuilder` 读取核心配置文件，构建出 `SqlSessionFactory`。
    3.  **创建会话**: `SqlSessionFactory` 创建 `SqlSession` 对象。
    4.  **执行操作**: `SqlSession` 通过底层的 Executor 执行器，将参数封装并操作数据库。
    5.  **返回结果**: 根据 Mapper 文件中的 `resultType` 或 `resultMap` 配置，将查询结果映射成Java对象并返回。

#### **3. 核心配置文件与 Mapper 映射**

*   **核心配置文件元素 (P15)**: 需了解常见配置元素的作用，如 `properties`, `settings`, `typeAliases`, `environments`, `mappers`。
*   **`<sql>` 元素 (P24)**: 掌握其用法。
    *   **定义**: 使用 `<sql id="片段ID">...</sql>` 定义可重用的SQL片段。
    *   **引用**: 使用 `<include refid="片段ID"/>` 在其他SQL语句中引用该片段。
*   **`<resultMap>` 元素 (P25-26)**: **极为重要和强大的功能**。
    *   **作用**: 手动定义数据库结果集与对象属性之间的映射规则，解决列名与属性名不一致的问题，以及处理复杂的级联关系。
    *   **与 `resultType` 的关系**: 两者**互斥**，不能同时使用。
    *   **核心配置 (P26, 图2-12)**:
        *   使用 `<resultMap id="自定义ID" type="目标对象类路径">` 替代 `<select>` 标签中的 `resultType`。
        *   内部使用 `<id>` 映射主键，使用 `<result>` 映射普通列。
        *   通过 `property` 属性指定对象属性名，`column` 属性指定数据库列名。

#### **4. MyBatis 工具类封装 (P28, 文件4)**

*   需理解工具类的封装逻辑，特别是 `getSqlSession()` 的实现过程：
    1.  **静态代码块 (static block)**: 类加载时执行，用于读取配置文件、创建 `SqlSessionFactory`。这是因为 `SqlSessionFactory` 是线程安全的，全局只需一个实例。
    2.  **`SqlSessionFactory` 声明**: 将 `SqlSessionFactory` 声明为 `private static` 成员变量。
    3.  **`getSqlSession()` 方法**: 提供一个 `public static` 方法，通过已创建的 `SqlSessionFactory` 调用 `openSession()`，返回一个新的 `SqlSession` 实例。

### **第二部分：Spring 框架 (第6-8章)**

*   **第三章内容**: 不考大题，小题部分根据“学习通”习题复习即可。

#### **1. Spring 概述 (P80)**

*   **Spring 定义**: 是一个分层的、一站式的**轻量级开源框架**。
*   **两大核心**:
    1.  **IoC (控制反转)**: Spring 的基础。将对象的创建权由开发者转移给 Spring IoC 容器。**必须能用自己的话解释其概念**（可类比“管家模式”），其核心是解决代码间的**解耦**问题。
    2.  **AOP (面向切面编程)**: Spring 的重要功能。动态地将通用代码（如日志、事务）织入到目标代码中，生成代理对象。

*   **IoC 与 DI 的关系 (P87-88)**: **核心概念，必须区分并理解**。
    *   **IoC (控制反转)**: 从**外部容器**的角度描述，容器控制了组件对象的创建和管理。
    *   **DI (依赖注入)**: 从**程序内部组件**的角度描述，容器在运行时动态地将一个对象所依赖的其他对象“注入”进去。**DI 是实现 IoC 的一种具体方式**。

#### **2. Bean 的配置与注入 (重点)**

*   **依赖注入的两种主要类型 (P88)**: **必须掌握**。
    1.  **构造注入 (Constructor Injection, P89)**:
        *   **XML配置**: 使用 `<constructor-arg>` 标签。`name` 属性对应构造方法参数名，`value` 或 `ref` 属性赋值。
    2.  **Setter 注入 (Setter Injection, P90)**:
        *   **XML配置**: 使用 `<property>` 标签。`name` 属性对应 `set` 方法的属性名，`value` 或 `ref` 属性赋值。
        *   **`value` 与 `ref` 的区别 (P91)**: `value` 用于注入基本类型和字符串；`ref` 用于注入其他已在容器中定义的 Bean。

*   **Bean 的基本配置 (P95, 表7-1)**: 掌握 `<bean>` 标签的四个核心属性。
    *   `id`: Bean的唯一标识符。
    *   `class`: Bean 的完整类路径。
    *   `name`: Bean 的别名，可有多个。
    *   `scope`: Bean 的作用域。

*   **Bean 的作用域 (P99, 7.4节)**: **重点掌握 `singleton` 和 `prototype` 的区别**。
    *   `singleton` (单例): **默认作用域**。在整个容器中只有一个实例。由 Spring 容器完整管理其生命周期（创建到销毁）。
    *   `prototype` (原型): 每次请求（获取）时都会创建一个新的实例。Spring 只负责创建，不负责后续销毁。

#### **3. 基于注解的 Bean 配置 (重点)**

*   **注解驱动开启 (P103, 文件7-17)**: 必须知道如何在 XML 配置文件中通过 `<context:component-scan base-package="..."/>` 来开启注解扫描。
*   **常用注解**: **必须熟练掌握以下所有注解**。
    *   **组件定义类**:
        *   `@Component`: 通用组件注解。
        *   `@Controller`: 用于表现层 (Controller)。
        *   `@Service`: 用于业务层 (Service)。
        *   `@Repository`: 用于持久层 (DAO)。
    *   **依赖注入类**:
        *   `@Autowired`: 按类型自动注入。
        *   `@Qualifier`: 配合 `@Autowired` 使用，按名称指定注入的 Bean。
        *   `@Resource`: 默认按名称注入，是 Java EE 规范的注解。

#### **4. AOP 面向切面编程 (第8章)**

*   **AOP 概念与术语 (P110)**: **必须能用自己的话串联术语来解释AOP**。核心思想是：将通用功能（**增强 Advice**）封装在**切面 (Aspect)** 中，然后通过配置，将其植入到目标对象特定方法（**连接点 JoinPoint**）的执行处（**切点 Pointcut**），最终生成代理对象的过程。
*   **JDK 动态代理 vs. CGLIB 动态代理 (P111-113)**: **此为新增考点，可能是简答题**。
    *   **实现方式**:
        *   JDK: 使用 `Proxy.newProxyInstance()` 方法。
        *   CGLIB: 使用 `Enhancer` 类。
    *   **代理对象**:
        *   JDK: **只能为实现了接口的类创建代理**。
        *   CGLIB: **可以为没有实现接口的普通类创建代理**（通过继承实现）。
    *   **核心接口/类**:
        *   JDK: 依赖 `InvocationHandler` 接口。
        *   CGLIB: 依赖 `MethodInterceptor` 接口。
*   **基于 XML 的 AOP 实现 (P120, 8.4节)**: **复习重点**。
    *   掌握如何在 XML 中配置切面 `<aop:aspect>`、切点 `<aop:pointcut>` 和不同类型的增强（如 `<aop:before>`, `<aop:after>` 等）。
    *   掌握如何通过 `<aop:config>` 启用 AOP 配置。

### **第三部分：Spring MVC (第10-11章)**

*   **第九章不考**。

#### **1. Spring MVC 概述与工作原理**

*   **Spring MVC 定义 (P140)**: 是一个基于 MVC 设计模式的 Web 框架，其核心是**前端控制器 `DispatcherServlet`**，用于接收请求并做出响应。
*   **核心工作原理 (P151, 图10-14)**: **必须掌握，是绝对的重点**。
    1.  `DispatcherServlet` 接收请求。
    2.  `DispatcherServlet` 请求 **处理器映射器 (HandlerMapping)** 查找 `Handler`。
    3.  `DispatcherServlet` 请求 **处理器适配器 (HandlerAdapter)** 执行 `Handler`。
    4.  `Handler` 执行业务逻辑后返回 `ModelAndView`。
    5.  `DispatcherServlet` 将 `ModelAndView` 交给 **视图解析器 (ViewResolver)** 解析视图。
    6.  `DispatcherServlet` 渲染视图并响应给客户端。
    *   *记忆技巧*: 可类比去银行办业务的流程（找窗口 -> 窗口核对材料 -> 办理业务 -> 返回凭证）。

#### **2. Spring MVC 核心配置**

*   **前端控制器配置 (P153, 文件11-1)**: **必考重点**，可能以任何题型出现。
    *   **`web.xml` 配置**: 掌握 `<servlet>` 和 `<servlet-mapping>` 的配置。
        *   `<servlet-class>`: 指向 `org.springframework.web.servlet.DispatcherServlet`。
        *   `<init-param>`: 通过 `contextConfigLocation` 指定 Spring MVC 配置文件的位置。
        *   `<load-on-startup>`: 设置为大于0的数，使 Servlet 在应用启动时加载。
        *   `<url-pattern>`: 通常配置为 `/`，拦截所有请求。

#### **3. `@RequestMapping` 注解 (重点)**

*   **作用 (P157)**: 配合 `@Controller` 使用，将请求的 URL 映射到具体的处理器方法上。
*   **使用位置**: 可以标注在**类**上（作为一级路径），也可以标注在**方法**上（作为二级路径）。
*   **常用属性 (P157, 表11-1)**: **必须掌握各属性的作用**。
    *   `value` / `path`: 指定映射的 URL 路径。
    *   `method`: 指定请求的方法类型（如 `RequestMethod.GET`, `RequestMethod.POST`）。如果请求方法不匹配，会报 405 错误。
    *   `params`: 根据请求参数进行映射，用于进一步缩小映射范围（如 `params = "id"` 表示必须有id参数）。
    *   `headers`: 根据请求头进行映射。
