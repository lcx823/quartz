策略模式：
### 🔍 图表解读

1. **左侧 (Calculator)**：代表上下文 Context。
    
    - 通过**空心菱形连线**指向右侧接口，表示它“持有”（聚合）一个策略对象，但不负责具体的策略实现细节。
        
2. **右上方 (OperationStrategy)**：代表抽象接口 Strategy。
    
    - 定义了所有算法必须遵守的 doOperation 契约。
        
3. **下方三个类**：具体的策略实现 (Addition, Subtraction, Multiplication)。
    
    - 通过**虚线三角箭头**指向接口，表示“实现” (implements) 关系。
        
4. **左下方 (Main)**：作为客户端，演示了它依赖于 Calculator 并负责装配策略。

工厂方法模式：
### 🔍 图表解读

1. **左列（工厂体系）**：
    
    - ShapeFactory (Top)：定义了标准的“生产线”规范。
        
    - CircleFactory / RectangleFactory (Bottom)：具体的“车间”，分别实现接口。
        
2. **右列（产品体系）**：
    
    - Shape (Top)：定义了产品的“外观”规范。
        
    - Circle / Rectangle (Bottom)：实际生产出的产品。
        
3. **核心连线（横向虚线箭头）**：
    
    - CircleFactory --> Circle：表示该工厂专门生产这种产品。
        
    - RectangleFactory --> Rectangle：表示该工厂专门生产这种产品。
        
4. **开闭原则体现**：
    
    - 如果需要添加新图形（如三角形），只需在下方继续添加 TriangleFactory 和 Triangle 类，而无需修改上方任何接口或现有的类。
![[./assets/类图/file-20251229143815956.png|file-20251229143815956.png]]

抽象工厂模式：
### 🔍 图表解读

1. **颜色编码**：
    
    - 🟨 **黄色**：抽象工厂接口（规则制定者）。
        
    - 🟦 **蓝色**：抽象产品接口（产品标准）。
        
    - 🟩 **绿色**：Windows 家族（具体工厂 + 具体产品）。
        
    - 🟪 **紫色**：Mac 家族（具体工厂 + 具体产品）。
        
2. **纵向关系（实现）**：
    
    - 所有 **具体工厂** 都指向 GUIFactory。
        
    - 所有 **具体按钮** 都指向 IButton。
        
    - 所有 **具体输入框** 都指向 ITextField。
        
3. **横向关系（创建/家族）**：
    
    - WindowsFactory 向右指向绿色的产品，表明它负责生产这一行。
        
    - MacFactory 向右指向紫色的产品，表明它负责生产这一行。
        
4. **Application (Client)**：
    
    - 位于顶部红色便签。
        
    - 它只依赖于**第一行**（抽象接口），完全不知道下面具体类的存在（符合依赖倒置原则）。
