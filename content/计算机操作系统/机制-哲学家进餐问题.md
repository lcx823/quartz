# 机制-哲学家进餐问题

哲学家进餐问题 (Dining Philosophers Problem) 是由Dijkstra提出的一个经典同步问题，用于说明多进程并发时可能出现的[[概念-死锁]]问题。

## 问题描述
- 五个哲学家围坐在一张圆桌旁，每人面前有一只碗，每两位哲学家之间放着一根筷子。
- 哲学家的活动是交替地思考和进餐。
- 哲学家进餐时，必须同时拿起他左右两边的两根筷子。

## 1. 利用记录型信号量解决
- **基本思路**: 将每根筷子看作一个临界资源，用一个互斥信号量表示。
  ```c
  semaphore chopstick = {1, 1, 1, 1, 1};

- **哲学家 i 的活动**:

  codeC

  ```
  do {
      // think
      wait(chopstick[i]);              // 拿起左边的筷子
      wait(chopstick[(i + 1) % 5]);   // 拿起右边的筷子
      // eat
      signal(chopstick[i]);              // 放下左边的筷子
      signal(chopstick[(i + 1) % 5]);   // 放下右边的筷子
  } while(TRUE);
  ```

- **问题**: 这个解法可能导致**死锁**。如果所有五个哲学家同时拿起他们左边的筷子，那么他们将永远等待右边的筷子，形成循环等待。

## 2. 死锁的解决方案

1. **至多只允许四位哲学家同时进餐**: 在进入房间时增加一个 wait 操作，离开时 signal。
2. **同时拿起两根筷子**: 使用[[机制-信号量|AND型信号量]]，将两个 wait 操作合并为一个原子操作 Swait(chopstick[i], chopstick[(i + 1) % 5])。这是最简洁的解决方法。
3. **规定奇偶号哲学家拿筷子顺序不同**:
   - 奇数号哲学家：先拿左边，后拿右边。
   - 偶数号哲学家：先拿右边，后拿左边。
     这样可以打破循环等待条件。

## 3. 利用管程解决

- **思路**: 使用[[机制-管程]]来控制筷子的拿起与放下。一个哲学家只有在左右两根筷子都可用时才被允许拿起筷子。
- **数据结构**:
  - state[5]: 记录每个哲学家的状态（思考、饥饿、进餐）。
  - condition self[5]: 每个哲学家在饥饿但不能拿到筷子时，在此条件变量上等待。
- **过程**:
  - pickup(i): 哲学家 i 试图拿起筷子。
  - putdown(i): 哲学家 i 放下筷子。
  - test(i): 检查哲学家 i 是否可以进餐，如果可以则唤醒他。
