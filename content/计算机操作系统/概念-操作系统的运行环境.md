# 概念-操作系统的运行环境

操作系统的正常运行需要特定的硬件支持，并构建了内核、双重工作模式、中断与异常等核心运行机制。

## 1. 硬件支持
- **基本构成**: 现代计算机系统由一个或多个CPU、若干设备控制器通过公用总线连接，共享内存。
- **引导程序 (Bootstrap Program)**: 计算机启动时运行的初始程序，通常位于固件（ROM或EEPROM）中。它的核心任务是**定位[[概念-操作系统内核]]并将其加载到内存中**开始执行。

## 2. [[概念-操作系统内核]]
- **定义**: 操作系统中一直运行在计算机上的核心程序。
- **功能**:
  - **支撑功能**: 中断处理、时钟管理、原语操作（原子操作）。
  - **资源管理功能**: [[概念-操作系统的主要功能|进程管理、存储器管理、设备管理]]中运行频率较高或与硬件紧密相关的部分。
- **目的**: 将核心模块常驻内存，一方面是为了保护它们不被应用程序破坏，另一方面是为了提高OS的运行效率。

## 3. [[机制-处理机的双重工作模式]]
- **目的**: 为了确保OS的正确运行，必须区分OS代码和用户代码的执行，硬件提供了双重工作模式支持。
- **模式**:
  - **内核态 (Kernel Mode)**: 也称管态、系统态。OS代码在此模式下运行，可以执行所有指令，包括**特权指令**。
  - **用户态 (User Mode)**: 也称目态。用户程序在此模式下运行，只能执行**非特权指令**。
- **切换**:
  - 计算机启动时处于内核态。
  - 用户程序请求OS服务时（通过[[概念-系统调用]]），会从用户态切换到内核态。
  - OS将控制权交还给用户程序前，会从内核态切换到用户态。
- **硬件支持**: 通过一个**模式位 (mode bit)** 来表示当前模式（如：内核态为0，用户态为1）。

## 4. [[机制-中断与异常]]
- **定义**: 现代操作系统是中断驱动的。事件的发生通过中断或陷阱来通知OS。
- **中断 (Interrupt)**: 由硬件（如I/O设备）通过系统总线发送信号给CPU来触发，也称为**外中断**。
- **陷阱/异常 (Trap/Exception)**: 由软件引起的中断，源于程序出错（如除零）或用户程序的特定请求（执行[[概念-系统调用]]），也称为**内中断**。
- **中断处理**:
  - CPU被中断后，会停止当前工作，转到固定位置执行。
  - 该位置通常包含中断处理程序的地址，这些地址集合构成了**中断向量表**。
