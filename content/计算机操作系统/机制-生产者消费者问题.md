# 机制-生产者消费者问题

生产者-消费者问题 (Producer-Consumer Problem) 是一个经典的进程同步问题，描述了一组生产者进程和一组消费者进程通过一个有限大小的缓冲区共享数据。

- **生产者**: 生产产品并放入缓冲区。
- **消费者**: 从缓冲区取出产品并消费。
- **约束**:
  1. 缓冲区满时，生产者必须等待。
  2. 缓冲区空时，消费者必须等待。
  3. 多个生产者和消费者对缓冲区的访问必须是互斥的。

## 1. 利用记录型信号量解决
- **信号量定义**:
  ```c
  // 假设缓冲区大小为 n
  semaphore mutex = 1;      // 互斥信号量，用于保护缓冲区
  semaphore empty = n;      // 同步信号量，表示空缓冲区的数量
  semaphore full = 0;       // 同步信号量，表示满缓冲区的数量
  ```

- **生产者伪代码**:

  codeC

  ```
  producer() {
      while(1) {
          produce an item;
          wait(empty);     // 获取一个空缓冲区
          wait(mutex);     // 进入临界区
          add item to buffer;
          signal(mutex);     // 离开临界区
          signal(full);      // 增加一个满缓冲区
      }
  }
  ```

- **消费者伪代码**:

  codeC

  ```
  consumer() {
      while(1) {
          wait(full);      // 获取一个满缓冲区
          wait(mutex);     // 进入临界区
          remove item from buffer;
          signal(mutex);     // 离开临界区
          signal(empty);     // 增加一个空缓冲区
          consume the item;
      }
  }
  ```

> **注意**: wait(empty)/wait(full) 和 wait(mutex) 的顺序不能颠倒。如果 wait(mutex) 在前，当缓冲区满（或空）时，进程会持有互斥锁并阻塞，导致其他进程也无法进入临界区释放资源，从而引发[[概念-死锁]]。

## 2. 利用管程解决

通过创建一个producerconsumer管程来封装缓冲区和同步操作。

- **管程内部**:
  - **共享数据**: buffer, in, out, count
  - **条件变量**: notfull, notempty
  - **过程**: put(item), get(item)
- **生产者**: 调用 PC.put(item)
- **消费者**: 调用 PC.get(item)

codeC

```c
monitor producerconsumer {
    item buffer[N];
    int in, out, count;
    condition notfull, notempty;

    public:
    void put(item x) {
        if (count >= N) cwait(notfull);
        buffer[in] = x;
        in = (in + 1) % N;
        count++;
        csignal(notempty);
    }

    void get(item &x) {
        if (count <= 0) cwait(notempty);
        x = buffer[out];
        out = (out + 1) % N;
        count--;
        csignal(notfull);
    }
}
```
