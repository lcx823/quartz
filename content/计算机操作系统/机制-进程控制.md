# 机制-进程控制

进程控制是进程管理中最基本的功能，负责[[概念-进程]]的整个生命周期管理，一般由OS内核中的**原语**来实现。

## 1. 进程的创建
- **引起创建的事件**:
  - **用户登录**: 在分时系统中为登录用户创建进程。
  - **作业调度**: 在批处理系统中为调入内存的作业创建进程。
  - **提供服务**: 系统为响应用户请求而创建进程（如打印进程）。
  - **应用请求**: 用户进程创建子进程以实现并发。
- **创建过程 (创建原语)**:
  1.  **申请空白PCB**: 为新进程分配唯一的标识符并获取一个空的[[概念-进程控制块 (PCB)]]。
  2.  **为新进程分配资源**: 分配内存、文件、I/O设备等。
  3.  **初始化PCB**: 初始化标识符、处理机状态（PC指向入口）、控制信息（设为就绪态）。
  4.  **插入就绪队列**: 将新进程插入就绪队列，等待调度。

## 2. 进程的终止
- **引起终止的事件**:
  - **正常结束**: 进程任务完成，自行请求终止。
  - **异常结束**: 发生错误，如越界、非法指令、I/O错误等。
  - **外界干预**: 操作员或父进程请求终止，或因父进程终止而被终止。
- **终止过程 (终止原语)**:
  1.  **检索PCB**: 根据进程标识符找到其PCB。
  2.  **终止执行**: 若进程处于执行状态，立即停止，置调度标志。
  3.  **终止其所有子孙进程**: 防止产生不可控的进程。
  4.  **归还资源**: 将其拥有的所有资源归还给父进程或系统。
  5.  **移出PCB**: 将其PCB从所在队列中移出。

## 3. 进程的阻塞与唤醒
- **阻塞 (Block)**: 是进程自身的一种**主动行为**。
  - **引起阻塞的事件**: 请求共享资源失败、等待操作完成、新数据未到达等。
  - **阻塞原语过程**: 立即停止执行 -> 修改PCB状态为阻塞 -> 插入相应事件的阻塞队列 -> 转调度程序。
- **唤醒 (Wakeup)**: 由相关进程（如完成I/O的进程）调用的**被动行为**。
  - **唤醒原语过程**: 从阻塞队列中移出 -> 修改PCB状态为就绪 -> 插入就绪队列。
> `block`原语和`wakeup`原语必须成对使用，否则进程会永久阻塞。

## 4. 进程的挂起与激活
- **挂起 (Suspend)**: 将进程从内存调至外存。
  - **挂起原语过程**: 检查进程状态，若为活动（就绪/阻塞），则改为静止（静止就绪/静止阻塞）。若进程正在执行，则重新调度。
- **激活 (Active)**: 将进程从外存调入内存。
  - **激活原语过程**: 将进程调入内存，检查其状态，若是静止就绪/阻塞，则改为活动就绪/阻塞。
