# 机制-处理机的双重工作模式

为了保护[[概念-操作系统内核]]不被用户程序破坏，并确保整个系统的稳定运行，现代CPU硬件都提供了至少两种独立的工作模式，即双重工作模式。

## 1. 两种模式
由硬件中的一个**模式位 (Mode Bit)** 来区分当前的工作状态。
1.  **内核态 (Kernel Mode)**
    - **别名**: 管态、系统态、特权态。
    - **运行代码**: [[概念-操作系统]]内核代码。
    - **权限**: 拥有最高权限，可以执行计算机系统中的**所有指令**，包括**特权指令**和**非特权指令**。
2.  **用户态 (User Mode)**
    - **别名**: 目态。
    - **运行代码**: 用户应用程序。
    - **权限**: 权限受限，只能执行**非特权指令**。如果试图执行特权指令，硬件会产生一个陷阱（trap），将控制权强制转交给操作系统。

## 2. 特权指令
- **定义**: 关系到系统全局、可能会引起损害的机器指令。例如：
  - I/O操作指令。
  - 中断屏蔽指令。
  - 修改程序状态字(PSW)的指令。
  - 访问和修改时钟、界限寄存器等特殊寄存器的指令。

## 3. 模式切换
- **用户态 -> 内核态**: 这是从用户程序进入OS内核的**唯一途径**。当用户程序需要请求OS服务时，会执行一个**[[概念-系统调用]]**，这会引发一个陷阱（软中断），CPU硬件自动将模式位从用户态切换到内核态，并将控制权交给OS内核中相应的处理程序。
- **内核态 -> 用户态**: 当OS完成了对用户请求的服务后，在将控制权返回给用户程序之前，会执行一条特权指令，将模式位从内核态切换回用户态。
