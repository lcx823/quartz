# 机制-硬件同步机制

为了克服纯[[机制-软件同步机制|软件同步机制]]的复杂性和局限性，许多计算机提供了特殊的硬件指令来高效地解决[[概念-临界资源与临界区|临界区]]问题。这些指令通常是**原子操作**，即执行过程不可中断。

## 1. 关中断
- **原理**: 在进入临界区之前关闭中断，完成临界区操作并退出后，再开启中断。在关中断期间，不会发生进程切换，从而保证了互斥。
- **缺点**:
  - 滥用关中断权力可能导致严重后果。
  - 关中断时间过长会影响系统效率。
  - **不适用于多CPU系统**，因为在一个CPU上关中断无法阻止其他CPU上的进程访问临界区。

## 2. Test-and-Set (TS) 指令
- **原理**: 这是一条原子指令，用于测试并设置一个锁变量（通常是布尔型 `lock`）。
- **伪代码**:
  ```c
  boolean TS(boolean *lock) {
      boolean old;
      old = *lock;
      *lock = TRUE;
      return old;
  }

- **使用方法**:

  codeC

  ```
  // lock 初始值为 FALSE
  while (TS(&lock)); // 循环测试，直到 lock 为 FALSE
  // 临界区
  lock = FALSE; // 释放锁
  ```

  当 TS 返回 FALSE 时，表示锁是打开的，进程可以进入临界区并同时将锁关闭。

## 3. Swap 指令 (XCHG 指令)

- **原理**: 这是一条原子指令，用于交换两个变量（通常是一个全局锁变量和一个进程局部变量）的内容。

- **伪代码**:

  codeC

  ```
  void swap(boolean *a, boolean *b) {
      boolean temp;
      temp = *a;
      *a = *b;
      *b = temp;
  }
  ```

- **使用方法**:

  codeC

  ```
  // 全局锁 lock 初始值为 FALSE
  // 进程局部变量 key
  key = TRUE;
  do {
      swap(&lock, &key);
  } while (key != FALSE); // 当交换后 key 变为 FALSE，表示获得了锁
  // 临界区
  lock = FALSE; // 释放锁
  ```

> 硬件同步机制虽然高效，但通常会导致进程**忙等**，不符合“让权等待”原则，浪费CPU资源。它们是构建更高级同步工具（如[[机制-信号量]]）的基础。
