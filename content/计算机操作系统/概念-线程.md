# 概念-线程

在操作系统中引入线程，是为了减少程序在并发执行时所付出的时空开销，使OS具有更好的并发性。线程是比[[概念-进程]]更小的能独立运行的基本单位。

## 线程与进程的比较

| 比较维度 | 进程 (Process) | 线程 (Thread) |
| :--- | :--- | :--- |
| **基本单位** | 拥有资源和独立调度的基本单位 | 独立调度的基本单位 |
| **拥有资源** | 可拥有独立的地址空间和系统资源 | 几乎不拥有资源，共享所属进程的资源 |
| **并发性** | 进程之间可以并发执行 | 同一进程内的多个线程可并发，不同进程的线程也可并发 |
| **独立性** | 独立性强，地址空间受保护 | 独立性弱，可访问同一进程的地址空间和资源 |
| **系统开销** | 创建、撤销、切换的开销大 | 创建、撤销、切换的开销小，被称为**轻型进程(LWP)** |
| **多处理机支持** | 单线程进程只能在一个处理机上运行 | 多线程进程的多个线程可分配到多个处理机上并行运行 |

## 线程的状态和线程控制块(TCB)
- **线程状态**: 与进程类似，线程也具有**执行、就绪、阻塞**三种基本状态。
- **线程控制块 (Thread Control Block, TCB)**: 系统为每个线程配置一个TCB，用于控制和管理线程。
  - **TCB内容**: 线程标识符、一组寄存器(PC,状态寄存器等)、线程状态、优先级、线程专有存储区、堆栈指针等。
