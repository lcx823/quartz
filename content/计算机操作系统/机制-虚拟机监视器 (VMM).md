# 机制-虚拟机监视器 (VMM)

虚拟机监视器 (Virtual Machine Monitor, VMM)，也称为**Hypervisor**，是[[概念-虚拟化]]技术中的核心软件层，负责创建和管理[[概念-虚拟化|虚拟机(VM)]]，并对底层物理硬件资源进行模拟、隔离与共享。

## 1. VMM的类型
### a. Type 1 VMM (裸金属架构)
- **结构**: VMM直接运行在物理硬件之上，它本身就是一个精简的操作系统。客户机OS运行在VMM之上。
- **特点**:
  - **高性能**: 直接控制硬件，路径短，开销小。
  - **高安全性与隔离性**: 架构简单，受攻击面小，不易受主机OS漏洞的影响。
- **应用**: 主要用于服务器和数据中心环境。
- **例子**: VMware ESXi, Xen, Microsoft Hyper-V, KVM (内核虚拟机)。

### b. Type 2 VMM (寄居架构)
- **结构**: VMM作为一个应用程序运行在传统的主机操作系统（如Windows, Linux）之上。
- **特点**:
  - **使用便捷**: 安装和卸载方便，就像一个普通软件。
  - **性能较低**: I/O和内存访问需要经过“客户机OS -> VMM -> 主机OS -> 硬件”的多层转换，性能开销大。
  - **安全性较低**: 依赖于主机OS的安全性。
- **应用**: 主要用于桌面虚拟化、开发和测试环境。
- **例子**: VMware Workstation, VirtualBox, Parallels Desktop。

## 2. KVM (内核虚拟机)
- **KVM (Kernel-based Virtual Machine)** 是一种特殊的Type 1 VMM。
- 它通过加载内核模块的方式，将**Linux内核本身**转变为一个Hypervisor。
- 它利用了Linux内核成熟的内存管理、进程调度、设备驱动等功能，同时结合了[[机制-CPU虚拟化|硬件辅助虚拟化]]技术，实现了高性能的虚拟化。
