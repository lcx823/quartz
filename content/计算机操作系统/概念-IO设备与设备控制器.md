# 概念-I/O设备与设备控制器

I/O设备由执行操作的机械部分和执行控制的电子部件组成。电子部件通常被称为**设备控制器**或**适配器**。

## 1. I/O设备
### a. 设备分类
- **按使用特性**:
  - **存储设备**: 外存/辅存，如磁盘、磁带。
  - **I/O设备**:
    - 输入设备：键盘、鼠标、扫描仪。
    - 输出设备：打印机、显示器。
    - 交互式设备：触摸屏。
- **按传输速率**:
  - **低速设备**: 键盘、鼠标。
  - **中速设备**: 打印机。
  - **高速设备**: 磁盘、光盘。

## 2. 设备控制器
- **功能**: CPU与I/O设备之间的接口，负责控制一个或多个I/O设备，实现数据交换，把CPU从繁杂的设备控制中解脱出来。
- **主要功能**:
  - **接收和识别命令**: 如`read`, `write`。
  - **数据交换**: 实现CPU、控制器和设备之间的数据传输。
  - **标志和报告设备的状态**: 通过状态寄存器供CPU查询。
  - **地址识别**: 识别自己所控制的设备地址。
  - **数据缓冲**: 内置缓冲区以缓和速度不匹配问题。
  - **差错控制**: 检测I/O过程中的错误。
- **组成**:
  - **与CPU的接口**: 数据线、地址线、控制线。
  - **与设备的接口**: 连接物理设备。
  - **I/O逻辑**: 负责执行CPU发来的命令，控制设备工作。

## 3. 内存映像I/O (Memory-Mapped I/O)
- **概念**: 一种将设备控制器的寄存器映射到内存地址空间中的技术。
- **特点**: CPU可以像访问普通内存单元一样，使用标准的`load/store`指令来访问设备寄存器，而无需专用的I/O指令。
- **优点**: 简化了I/O设备的编程。

## 4. I/O通道 (I/O Channel)
- **引入目的**: 进一步减少CPU对I/O的干预。通道是一种特殊的处理机，能执行自己的**通道程序**来管理一系列I/O操作。
- **工作流程**: CPU只需向通道发送一条I/O指令，通道便会从内存中取出并执行整个通道程序，完成后才向CPU发送中断信号。
- **通道类型**:
  - **字节多路通道**: 按字节交叉方式分时为多个低速设备服务。
  - **数组选择通道**: 一段时间内独占地为一个高速设备传送一个数据块。
  - **数组多路通道**: 结合了前两者优点，分时为多个高速设备以数据块为单位进行传输。
