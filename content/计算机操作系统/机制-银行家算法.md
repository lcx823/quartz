# 机制-银行家算法

银行家算法（Banker's Algorithm）是由Dijkstra提出的一个著名的**死锁避免**算法。它通过在资源分配前进行安全性检查，确保系统始终处于**安全状态**，从而避免进入可能导致[[概念-死锁|死锁]]的不安全状态。

## 1. 核心思想
- **安全状态**: 指系统能按某种进程推进顺序（称为**安全序列**），为每个进程分配其所需的资源，直至满足每个进程对资源的最大需求，使每个进程都可以顺利完成。
- **算法流程**:
  1. 进程在进入系统时，必须申明其在运行过程中可能需要的每种资源类型的最大数量。
  2. 当进程请求一组资源时，系统首先检查是否有足够的可用资源分配给它。
  3. 若有，系统**试探性地**将资源分配给该进程，并执行**安全性算法**，检查分配后的新状态是否仍然是安全状态。
  4. 如果是安全状态，则正式分配资源；如果不是安全状态，则撤销此次试探性分配，让进程等待。

## 2. 数据结构
- `Available`: 可利用资源向量，表示每类资源的可用数量。
- `Max`: 最大需求矩阵，定义了每个进程对每类资源的最大需求。
- `Allocation`: 分配矩阵，定义了当前已分配给每个进程的每类资源数。
- `Need`: 需求矩阵，表示每个进程尚需的各类资源数。`Need = Max - Allocation`。

## 3. 安全性算法
1.  设置工作向量 `Work = Available` 和完成向量 `Finish` (初始为 `false`)。
2.  从进程集合中寻找一个能满足以下条件的进程 `Pi`:
    - `Finish[i] == false`
    - `Need[i] <= Work`
3.  若找到，则假定 `Pi` 执行完成并释放资源，更新 `Work = Work + Allocation[i]`，并设置 `Finish[i] = true`，然后返回步骤2。
4.  若找不到这样的进程，则检查 `Finish` 向量。如果所有 `Finish[i]` 都为 `true`，则系统处于安全状态；否则，系统处于不安全状态。
