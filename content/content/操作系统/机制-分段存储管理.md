# 机制-分段存储管理

分段存储管理方式是为了更好地满足用户在编程和使用上的多方面需求而引入的一种离散分配方式。它更符合用户对信息的逻辑组织。

## 1. 引入目的
1.  **方便编程**: 用户可将程序按逻辑关系划分为若干个段（如主程序段、子程序段、数据段、栈段），每个段有自己的名字和长度，从0开始编址。
2.  **[[./机制-文件共享|信息共享]]**: 以信息的逻辑单位（段）为基础实现共享，比[[./机制-分页存储管理|机制-分页存储管理]]更方便、更有效。
3.  **[[./机制-文件保护|信息保护]]**: 可以对信息的逻辑单位（段）进行保护，如将代码段设为只执行，数据段设为可读写。
4.  **动态链接**: 以段（目标程序）为单位进行运行时链接非常自然。
5.  **动态增长**: 可以很方便地处理某些段（如数据段、栈段）在运行过程中的长度变化。

## 2. 基本原理
- **分段**: 将作业的地址空间按逻辑关系划分为若干个**段**，每个段的长度可不同。
- **逻辑地址结构**: 由**段号 (S)** 和**段内地址 (d)** (位移量)组成，是一个二维地址结构。
- **段表 (Segment Table)**:
  - **作用**: 实现从逻辑段到物理内存区的映射。
  - **结构**: 系统为每个进程建立一张段表。段表中的每个**段表项**记录了该段在内存中的**起始地址（基址）**和**段的长度**。
  - **存放**: 系统设置一个**段表寄存器**，存放该进程段表的起始地址和长度。

## 3. 地址变换机构
1.  **分离地址**: 从逻辑地址中分离出段号 `S` 和段内地址 `d`。
2.  **检查越界 (段号)**: 将段号 `S` 与段表寄存器中的段表长度 `TL` 比较，若 `S >= TL` 则产生越界中断。
3.  **查找段表**:
    - 段表项地址 = 段表起始地址 + 段号 `S` × 段表项长度。
    - 从计算出的地址中读取段表项，获得段的基址和段长 `SL`。
4.  **检查越界 (段内地址)**: 将段内地址 `d` 与段长 `SL` 比较，若 `d >= SL` 则产生越界中断。
5.  **形成物理地址**: 物理地址 = 段基址 + 段内地址 `d`。

## 4. 分页与分段的主要区别

| 特性 | 分页 (Paging) | 分段 (Segmentation) |
| :--- | :--- | :--- |
| **单位性质** | 信息的**物理单位** | 信息的**逻辑单位** |
| **目的** | 提高内存利用率，是**系统行为** | 满足用户需求，是**用户行为** |
| **大小** | 页面大小**固定**且由系统决定 | 段的长度**不固定**，由程序决定 |
| **地址空间** | **一维**地址空间 | **二维**地址空间 |
| **对用户** | **透明** | **可见** |
