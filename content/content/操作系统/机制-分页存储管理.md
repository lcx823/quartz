# 机制-分页存储管理

分页存储管理是一种**离散分配**的存储管理方式，它旨在解决[[./机制-连续分配存储管理|连续分配]]方式产生的外部碎片问题，从而提高内存利用率。

## 1. 基本方法
- **页面 (Page)**: 将进程的**逻辑地址空间**划分为若干个大小固定的区域。
- **物理块 (Frame)**: 将**物理内存空间**也划分为与页面大小相同的区域。
- **分配**: 以块为单位为进程分配内存，进程的任一页可以装入内存的任一物理块中，各页面在内存中无需连续。
- **页内碎片**: 进程的最后一页通常装不满一个物理块，这部分未被利用的空间称为页内碎片或内碎片。

## 2. 地址结构与页表
- **逻辑地址结构**: 由**页号 (P)** 和**页内地址 (d)** (位移量)组成。
- **页表 (Page Table)**:
  - **作用**: 实现从页号到物理块号的地址映射。
  - **结构**: 系统为每个进程建立一张页表。页表中的每个**页表项 (Page Table Entry, PTE)** 记录了逻辑页号对应的物理块号。
  - **存放**: 页表通常存放在内存中。系统设置一个**页表寄存器 (PTR)**，存放该进程页表的起始地址和长度。

## 3. 地址变换机构
这是一个硬件机构，负责将逻辑地址转换为物理地址。
1.  **分离地址**: 将逻辑地址分离为页号 `P` 和页内地址 `d`。
2.  **检查越界**: 将页号 `P` 与页表寄存器中的页表长度 `TL` 比较，若 `P >= TL` 则产生越界中断。
3.  **查找页表**:
    - 页表项地址 = 页表起始地址 + 页号 `P` × 页表项长度。
    - 从计算出的地址中读取页表项，获得物理块号 `b`。
4.  **形成物理地址**: 物理地址 = 物理块号 `b` × 页面大小 + 页内地址 `d`。

## 4. 具有快表的地址变换机构
- **问题**: 由于页表存放在内存中，每次访问数据需要访问**两次内存**（一次访页表，一次访数据），导致速度降低近一半。
- **解决方案**: 增设一个具有并行查询能力的高速缓冲寄存器，称为**快表 (Translation Look-aside Buffer, TLB)** 或**联想寄存器**。
- **工作原理**:
  1.  地址变换时，先在快表中查找页号。
  2.  **快表命中 (Hit)**: 如果找到，直接从中读出物理块号，形成物理地址。只需访问一次内存（取数据）。
  3.  **快表未命中 (Miss)**: 如果未找到，则需访问内存中的页表，找到物理块号后，**同时将该页表项写入快表**，以备下次使用。需要访问两次内存。
- **性能**: 基于[[./概念-局部性原理|概念-局部性原理]]，快表的命中率通常很高（90%以上），极大地提高了地址变换的效率。

## 5. 多级页表与反置页表
- **问题**: 对于逻辑地址空间非常大的系统（如64位系统），单级页表会变得极其庞大，占用大量连续内存。
- **解决方案**:
  - **两级/多级页表**: 将页表本身也进行分页，用一个**外层页表**来记录各个页表页面的存放位置。这样，页表就可以离散地存放在内存中。
  - **反置页表**: 不为每个进程建立页表，而是为系统中的每个**物理块**建立一个页表项，记录该物理块中存放的是哪个进程的哪个页面。这种方式大大节省了内存空间，但查找过程变得复杂，通常需要借助哈希表。
