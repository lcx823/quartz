# 机制-Linux进程同步

Linux内核提供了完备的同步方法来避免多个进程并发访问同一临界资源。并发的来源主要有中断处理、内核态抢占和多处理机并发。

## 1. 原子操作
- **定义**: 保证指令以原子的方式执行，执行过程不被打断。它是其他同步方法的基础。
- **实现**:
  - **`lock`前缀**: 在多CPU下，`lock`前缀的汇编指令会锁定内存总线，保证其后指令的原子性。
  - **`atomic_t`类型**: 封装了一系列原子操作，如 `atomic_inc()`（原子加1）。

## 2. 自旋锁 (Spin Lock)
- **定义**: Linux内核中最常用的锁，尤其是在多处理机系统中。它是一种**忙等**机制。
- **原理**: 当一个进程需要获取锁而被占用时，它会持续循环测试锁的状态（自旋），直到锁被释放。它不会让进程进入睡眠，避免了上下文切换的开销。
- **适用场景**: 临界区非常小，锁被持有的时间非常短。
- **特点**: 保持自旋锁期间是**不可抢占**的。
- **使用**:
  ```c
  spin_lock(&my_lock);
  // 临界区
  spin_unlock(&my_lock);

## 3. 信号量 (Semaphore)

- **定义**: 一种可以使进程睡眠的同步机制。

- **原理**: 当进程无法获取信号量时，会立即释放CPU，并进入睡眠状态，被放入等待队列。当信号量被释放时，再唤醒等待队列中的进程。

- **适用场景**: 临界区较大，锁被持有的时间较长或不确定。

- **特点**: 保持信号量期间是**可以被抢占**的。

- **使用**:

  codeC

  ```
  // down_interruptible() 尝试获取信号量，若失败则睡眠
  if (down_interruptible(&my_sem)) { /* error */ }
  // 临界区
  up(&my_sem); // 释放信号量
  ```

## 4. 互斥锁 (Mutex)

- **定义**: 一种可以阻塞的强制互斥锁，是信号量的简化版。

- **行为**: 类似于使用计数为1的[[机制-信号量]]，但接口更简单，实现更高效。

- **使用**:

  codeC

  ```
  mutex_lock(&my_mutex);
  // 临界区
  mutex_unlock(&my_mutex);
  ```

## 5. 禁用中断

- **适用场景**: 在单处理机不可抢占系统中，主要的并发源是中断。通过在临界区前后禁用/使能中断，可以实现互斥。
- **注意**: 禁用中断的时间必须非常短，否则会严重影响系统响应外部中断的能力。
