# 机制-用户层的I/O软件

大部分I/O软件都位于操作系统内核中，但仍有一小部分功能由运行在用户空间的软件模块来完成。

## 1. 系统调用与库函数
- **[[./概念-系统调用|概念-系统调用]]**:
  - 是应用程序请求操作系统内核服务的**唯一途径**。
  - 用户程序通过系统调用间接调用内核中的I/O过程来操作设备，这个过程涉及从用户态到内核态的转换。

- **库函数**:
  - 许多语言（如C语言）提供了与系统调用功能相对应的库函数（如 `printf` 对应 `write` 系统调用）。
  - 库函数封装了系统调用的细节，为程序员提供了更方便、更友好的接口。
  - 库函数与用户程序链接在一起，运行在用户空间。

## 2. [[./机制-假脱机系统 (SPOOLing)|机制-假脱机系统 (SPOOLing)]]
- **定义**: SPOOLing (Simultaneous Peripheral Operations On-Line) 是一种将独占设备（如打印机）虚拟为可共享设备的技术。
- **实现**: 它是一个完整的、运行在内核之外的系统，通过在磁盘上建立输入井和输出井，利用专门的输入/输出进程来模拟脱机I/O，实现了外围设备与CPU操作的并行。
- **作用**: 是用户层I/O软件的一个典型例子，实现了[[./概念-虚拟存储器|虚拟设备]]功能。

## 3. 守护进程 (Daemon)
- **定义**: 在后台运行，等待处理特定事件或任务的进程。
- **在I/O中的应用**:
  - **打印机守护进程**: 监视打印队列（假脱机文件目录），当有打印任务时，将其发送给打印机。
  - **网络守护进程**: 监听网络端口，等待并处理网络连接请求。
- **特点**: 守护进程是允许访问特定独占设备的唯一进程，其他进程通过与守护进程通信来间接使用设备。```

# 机制-假脱机系统 (SPOOLing)

**假脱机技术 (SPOOLing, Simultaneous Peripheral Operations On-Line)** 是一种在联机情况下实现同时外围操作的技术，它利用磁盘作为高速缓冲，将一台物理上的独占设备（如打印机）虚拟成多台逻辑上的共享设备。

## 1. 组成
1.  **输入井和输出井**: 在磁盘上开辟的两个大存储区。输入井用于暂存从输入设备输入的数据，输出井用于暂存用户程序的输出数据。
2.  **输入缓冲区和输出缓冲区**: 在内存中开辟的缓冲区，用于缓和CPU与磁盘之间的速度矛盾。
3.  **输入进程和输出进程**:
    - **输入进程 (预输入进程)**: 模拟脱机输入，将数据从低速输入设备传送到磁盘输入井。
    - **输出进程 (缓输出进程)**: 模拟脱机输出，将数据从磁盘输出井传送到低速输出设备。
4.  **井管理程序**: 控制作业与磁盘井之间的信息交换。

## 2. 核心特点
1.  **提高了I/O速度**: 将对低速设备的I/O操作，转变为对高速磁盘（输入/输出井）的I/O操作，缓和了CPU与低速设备的速度矛盾。
2.  **将独占设备改造为共享设备**: 系统并未真正为进程分配物理设备，而是在磁盘井中为其分配了空间和请求表。多个进程可以“同时”请求使用独占设备。
3.  **实现了虚拟设备功能**: 对每个进程而言，它们都认为自己独占了一台逻辑设备，而实际上它们共享的是同一台物理设备。

## 3. 假脱机打印机系统示例
- 用户进程发出打印请求后，假脱机系统并不立即打印。
- 而是由**假脱机管理进程**在磁盘输出井中为该进程分配空间，存放打印数据，并创建一个打印请求表，挂入**假脱机文件队列**。
- 对于用户进程来说，打印操作“瞬间”完成。
- 真正的打印由**假脱机打印进程**（或**打印机守护进程**）在后台执行。当打印机空闲时，它会从假脱机文件队列中取出任务，将数据从输出井传送到打印机进行打印。
