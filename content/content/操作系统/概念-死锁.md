# 概念-死锁

## 1. 定义
死锁（Deadlock）是指一组进程中的每个进程都在等待仅由该组进程中的其他进程才能引发的事件，导致所有进程都无法向前推进，陷入无限期等待的状态。

> 如果一组进程中的每个进程都在等待仅由该组进程中的其他进程才能引发的事件发生，那么该组进程是死锁的。
> —— 第3章 3.5.3节

## 2. 产生死锁的必要条件
一个死锁的发生必须**同时**满足以下四个条件：
1.  **互斥条件**: 进程对所分配到的资源进行排他性使用。
2.  **请求和保持条件**: 进程已至少保持一个资源，但又提出新的资源请求，而该资源已被其他进程占用，此时请求进程被阻塞，但对自己已获得的资源保持不放。
3.  **不可抢占条件**: 进程已获得的资源在未使用完之前，不能被强行剥夺，只能在使用完后由自己释放。
4.  **循环等待条件**: 存在一个进程-资源的循环链，`P0`等待`P1`的资源，`P1`等待`P2`的资源，...，`Pn`等待`P0`的资源。

## 3. 产生死锁的原因
- **竞争不可抢占资源**: 多个进程对数量不足的不可抢占资源（如打印机、文件）的争夺。
- **竞争可消耗资源**: 如进程间通信时，多个进程都在等待接收对方的消息而形成循环。
- **进程推进顺序不当**: 进程在运行过程中，对资源申请和释放的顺序不当，可能导致进入不安全状态，进而引发死锁。

## 4. 死锁的处理方法
1.  **[[./机制-死锁预防|机制-死锁预防]]**: 通过破坏死锁产生的四个必要条件之一来预防死锁的发生。
2.  **死锁避免**: 在资源动态分配过程中，用算法（如[[./机制-银行家算法|机制-银行家算法]]）防止系统进入不安全状态。
3.  **[[./机制-死锁的检测与解除|死锁检测]]**: 允许系统进入死锁状态，通过检测算法及时发现死锁。
4.  **[[./机制-死锁的检测与解除|死锁解除]]**: 当检测到死锁后，采取措施（如终止进程、抢占资源）将系统从死锁状态中解脱出来。
