# 机制-软件同步机制

软件同步机制是指不依赖于特殊硬件指令，仅通过软件算法来解决[[./概念-临界资源与临界区|临界区]]互斥问题的机制。**Peterson解决方案**是其中一个经典的算法。

## Peterson解决方案
该方案适用于两个进程 `P0` 和 `P1` 交替执行临界区和剩余区的情况。

- **共享变量**:
  ```c
  int turn;       // 表示哪个进程可以进入临界区
  boolean flag; // 表示哪个进程准备进入临界区

- **进程 Pi 的算法结构**:

  codeC

  ```
  // 假设另一个进程为 Pj
  do {
      flag[i] = TRUE;
      turn = j;
      while (flag[j] && turn == j); // 忙等待
  
      // 临界区
      critical_section();
  
      // 退出区
      flag[i] = FALSE;
  
      // 剩余区
      remainder_section();
  } while (TRUE);
  ```

## 算法分析

- **满足互斥**: 通过 turn 和 flag 变量的巧妙配合，保证了两个进程不会同时进入临界区。如果两个进程同时请求，最终只有一个 turn 的赋值会生效，决定了谁先进入。
- **避免死锁**: 满足“空闲让进”和“有限等待”准则。如果只有一个进程请求进入，它不会被阻塞。如果两个都请求，它们不会互相无限期等待。
- **缺点**: 该算法使用了“忙等”的方式，不符合“让权等待”准unze，会浪费CPU时间。此外，由于现代计算机指令执行方式的复杂性，不能保证该方案在所有硬件上都能正确运行。
