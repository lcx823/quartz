# 概念-I/O系统的功能、模型与接口

I/O系统是操作系统的主要组成部分，负责管理I/O设备及相应的设备控制器。

## 1. I/O系统的基本功能
1.  **隐藏I/O设备细节**: 通过对I/O设备进行抽象，向上层提供统一的、简单的读写命令（如`read`, `write`），隐藏底层硬件的复杂性。
2.  **保证设备无关性**: 允许用户程序使用抽象的逻辑设备名来请求I/O操作，而不是具体的物理设备名。这使得程序不依赖于特定硬件，并支持I/O重定向。
3.  **提高处理机和I/O设备的利用率**: 尽可能让CPU和I/O设备并行操作。
4.  **对I/O设备进行控制**: 采用不同的[[./机制-IO设备的控制方式|控制方式]]（如中断、DMA）来驱动设备工作。
5.  **确保对设备的正确共享**: 对独占设备（如打印机）和共享设备（如磁盘）进行有效的分配与管理。
6.  **处理错误**: 在尽可能低的层次上处理I/O错误，对上层屏蔽错误细节。

## 2. I/O系统的层次结构与模型
I/O软件通常被组织成一个层次结构，以实现清晰的设计和良好的可移植性。
- **I/O软件层次**:
  1.  **[[./机制-用户层的IO软件|用户层I/O软件]]**: 提供用户接口，如库函数。
  2.  **[[./机制-与设备无关的IO软件|与设备无关的I/O软件]]**: 实现设备命名、保护、分配、缓冲等通用功能。
  3.  **[[./机制-设备驱动程序|机制-设备驱动程序]]**: 与硬件直接相关，负责驱动具体设备。
  4.  **[[./机制-中断与中断处理程序|机制-中断与中断处理程序]]**: 处理硬件中断，是整个I/O系统的基础。

- **I/O系统模型**:
  - **上接口 (I/O系统接口)**: 面向文件系统、虚拟存储器和用户进程。
  - **下接口 (软件/硬件接口)**: 面向各种设备控制器。

## 3. I/O系统接口
根据设备类型的不同，I/O系统提供不同的接口：
- **块设备接口**:
  - **对象**: 磁盘等以**数据块**为单位存取和传输的设备。
  - **特点**: 可寻址（随机访问），速率高，通常采用DMA方式。接口将磁盘的三维结构（柱面、磁头、扇区）抽象为线性的逻辑块号。
- **流设备接口 (字符设备接口)**:
  - **对象**: 键盘、打印机等以**字符 (字节)** 为单位存取和传输的设备。
  - **特点**: 不可寻址（顺序访问），速率较低，通常采用中断驱动方式。通过`get/put`操作进行数据交换。
- **网络通信接口**:
  - **对象**: 网络设备。
  - **特点**: 提供面向网络的通信功能，涉及网络协议和复杂的硬件。
