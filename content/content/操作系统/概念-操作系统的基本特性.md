# 概念-操作系统的基本特性

现代操作系统普遍具有四个基本特性：**并发、共享、虚拟、异步**。

## 1. 并发 (Concurrence)
- **定义**: 两个或多个事件在同一时间间隔内发生。在多道程序环境下，宏观上指有多个程序在同时运行，但在单处理机系统中，微观上这些程序是分时交替执行的。
- **与并行的区别**:
  - **并行 (Parallelism)**: 两个或多个事件在**同一时刻**发生，这需要多处理机系统的支持。
- **实现**: 通过引入[[./概念-进程|概念-进程]]和[[./概念-线程|概念-线程]]实现，是操作系统最重要的基本概念之一。

## 2. 共享 (Sharing)
- **定义**: 系统中的资源可供内存中多个并发执行的进程共同使用，也称为资源复用。
- **实现方式**:
  - **互斥共享方式**: 在一段时间内只允许一个进程访问的资源，称为[[./概念-临界资源与临界区|临界资源]]（如打印机）。系统需要提供机制来保证互斥访问。
  - **同时共享方式**: 允许在一段时间内由多个进程“同时”访问的资源（宏观上）。典型的例子是磁盘设备和可重入代码编写的文件。

> 并发和共享是多用户OS的两个最基本的特征，它们互为对方存在的条件。

## 3. 虚拟 (Virtual)
- **定义**: 通过某种技术将一个物理实体变为若干个逻辑上的对应物。OS中主要利用时分复用和空分复用技术实现虚拟。
- **实现技术**:
  - **时分复用技术**:
    - **虚拟处理机技术**: 利用多道程序设计技术，将一台物理处理机虚拟为多台逻辑上的处理机。
    - **虚拟设备技术**: 将一台物理I/O设备虚拟为多台逻辑上的I/O设备，如通过[[./机制-假脱机系统 (SPOOLing)|机制-假脱机系统 (SPOOLing)]]技术将打印机虚拟化。
  - **空分复用技术**:
    - **[[./概念-虚拟存储器|概念-虚拟存储器]]**: 利用存储器的空闲空间来存放其他程序，并通过分时复用内存的方式，在逻辑上扩大内存容量。

## 4. 异步 (Asynchronism)
- **定义**: 在多道程序环境下，由于资源等因素的限制，进程的执行通常不是“一气呵成”的，而是以“停停走走”的方式运行。进程以人们不可预知的速度向前推进。
- **原因**: 进程的执行受到资源分配、其他进程运行情况以及I/O操作等多种因素的制约，导致其执行速度和顺序具有不确定性。
- **结果**: 尽管存在异步性，但只要OS配置了完善的[[./机制-信号量|进程同步机制]]，在相同运行环境下，程序多次运行应能获得相同的结果（可再现性）。
